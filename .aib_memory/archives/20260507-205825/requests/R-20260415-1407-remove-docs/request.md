# Request

## Goal

The current 27 documents in `.aib_memory\docs` and their respective convention files should be removed and their function should be replaced by `.aib_memory\context.md` and `.aib_brain\conventions\context-convention.md`. The prompt `.aib_brain\prompts\aib-documentation.md` should be removed and the functionality to be covered by `.aib_brain\prompts\aib-context.md`. `.aib_memory\context.md` should be updated with each run of `.aib_brain\prompts\aib-implement.md`.  From `.aib_brain\prompts\aib-implement.md` should be removed execution of `.aib_brain\prompts\aib-documentation.md` as `.aib_brain\prompts\aib-documentation.md` should be removed as well. Update `.aib_brain\Concepts.md` and `.aib_brain\README.md` respectively. `.aib_brain\Product_Documentation.md` should be removed and the specifications in the docs - covered in `.aib_memory\context.md`

## Background

The AIB framework previously managed product knowledge through two parallel mechanisms: (1) a set of per-topic Markdown files under `.aib_memory/docs/` (27 registered in `references.md` plus 21 unregistered), governed by individual per-doc conventions and populated by `aib-documentation.md` and `aib-reverse-engineer.md`; and (2) a unified synthesis document `.aib_memory/context.md` generated by `aib-context.md`. Maintaining both structures creates duplication, increases the overhead of the `implement` workflow (which must call `aib-documentation.md` after every run), and requires maintaining 27+ per-doc convention files. A prior request introduced `context.md` and `aib-context.md` as a more streamlined path. This request eliminates the older per-doc approach entirely, consolidating product knowledge into the single unified `context.md` artefact.

## Scope
