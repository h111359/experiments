## Format 
 - No versioning of the analysis file
 - Do not include the author attribute in the header of the analysis
 - Generate a copy/paste ready code block with the text of the file. Inside the code block, escape all backticks or replace them with placeholders so the code block never breaks. Produce the entire document inside a single fenced code block without prematurely closing it. Ensure that list items that introduce code examples indent them correctly so Markdown does not break the fence.
 - Enclose the whole document with four backticks as the outer fence (````). 
Inside, you may use standard single or triple backticks normally. Do not use [[BT]].
 - When generating the code block, do NOT include any external hyperlinks, refs, citations, footnotes, preview links, or autogenerated reference markers such as ..., cite, or similar.
 - Inside the code block, produce only the pure document content.  
 - In the generated code block, DO NOT include any metadata header section such as:
	- Title:
	- Version:
	- Status:
	- Owner:
	- Applies-To:
	- Last-Updated:
	or any similar document level metadata.
Begin the document directly with the first normative section (e.g., “Purpose”) and exclude all header metadata entirely.
