## Goal 

Propose content for <FILE-PATH> convention so to cover the purpose as defined in the attached files `Concepts.md` and `Product_Documentation.md`. 

## Context 

Read the attached concepts and related documents specification. `Product_Documentation.md` contains the holistic idea for documents. You should find there the initial concept for the <FILE-PATH>. 

## Objectives
- propose text of the convention which strictly defines how the related file should be named, structured and operated.
- Align with best practices and most logical content of the document
- Do not overcomplicate - make it appropriate for AI reading and human verification
- Define all the aspects of proper and convenient format of the file with providing enough information to the human for easy decision taking

## Format 
 - No versioning of the file
 - Do not include the author attribute in the header
 - Do not include the location of the file - it will be defined elsewhere (don't mention in the convention)
 - **DO NOT** refer in any way the current framework. The convention should be portable (don't mention in the convention)
 - Generate a copy/paste ready code block with the text of the file. Inside the code block, escape all backticks or replace them with placeholders so the code block never breaks. Produce the entire document inside a single fenced code block without prematurely closing it. Ensure that list items that introduce code examples indent them correctly so Markdown does not break the fence.
 - Enclose the whole document with four backticks as the outer fence (````). 
Inside, you may use standard single or triple backticks normally. Do not use [[BT]].
 - When generating the code block, do NOT include any external hyperlinks, refs, citations, footnotes, preview links, or autogenerated reference markers such as ..., cite, or similar.
 - Inside the code block, produce only the pure document content.  
 - In the generated code block, DO NOT include any metadata header section such as:
	- Title:
	- Version:
	- Status:
	- Owner:
	- Applies-To:
	- Last-Updated:
	or any similar document level metadata.
Begin the document directly with the first normative section (e.g., “Purpose”) and exclude all header metadata entirely.
