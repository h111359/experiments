## 2. Domain Knowledge Essentials

- **AIB (AI Builder):** Minimal, file-first framework for specification-driven development. Owns `.aib_brain/` (framework assets) and `.aib_memory/` (workspace-specific runtime artifacts).

- **`.aib_memory/`:** Workspace-scoped memory folder holding requests, registers, references, and product documentation. After this change, it is also the home for transient action execution logs.

- **Action execution logs:** Transient UTF-8 flat-text files written by `menu.py` during interactive command execution. Named `aib-action-<YYYYMMDD>-<HHmmss>-<action-id>.log`. Excluded from VCS; session-specific.

- **Release version logs:** Markdown files named `version_vX.Y.Z_log.md` generated by `release_bookkeeping.py` via CI on every PR to `main`. Committed to VCS; retained indefinitely. Location unchanged by this request.

- **`logs/` (root):** Directory that currently holds both log categories. After this request, it continues to hold only release version logs. It must remain in VCS.

- **`.aib_memory/logs/`:** New home for action execution logs only. Created on demand by `_make_log_path()`.

- **Affected roles:** Developer (runs the menu, sees new log path), AIB Maintainer (owns `.aib_brain/`), CI/CD pipeline (unaffected — version log path unchanged).

- **Business processes touched:** BP-0002/BP-0004 indirectly (menu execution log path changes). BP-0005 (Release bookkeeping) is explicitly unaffected.

- **Acceptance impact:** After this change, no transient AIB action log will be created under `logs/`. Release log behaviour is unchanged.
