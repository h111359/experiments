# Request

## Goal

Enforce code commenting standards in every file generated or edited by the AI Automation Agent during the `implement` workflow. Achieve this by:

1. Creating a general coding-principles convention file (`.aib_brain/conventions/coding-general-convention.md`) that mandates comment requirements and code-quality best practices applicable to all programming languages.
2. Creating twelve language-specific convention files under `.aib_brain/conventions/` — one each for Python, DAX, SQL, HTML, JavaScript, CSS, React, Flask, Django, C#, Scala, and UI/UX design — each named `coding-<language>-convention.md`.
3. Updating `.aib_brain/prompts/aib-implement.md` so that the AI Automation Agent unconditionally loads the general coding convention at the start of every `implement` run, and additionally loads the matching language-specific convention when the files being created or edited match that language's file extensions.

## Background

Currently the AIB `implement` prompt (`aib-implement.md`) contains no reference to code-commenting standards or language-specific coding conventions. Code generated by the AI Automation Agent may lack comments, docstrings, or consistent style. The `.aib_brain/conventions/` folder holds 36 convention files covering product documentation and AIB structural artifacts but zero coding-style conventions.

This gap means that developers reviewing AI-generated code must infer intent without inline documentation, which increases review time and reduces maintainability.

## Scope

- Create `.aib_brain/conventions/coding-general-convention.md`: normative rules for file-level headers, function/method/class docstrings, inline comments on non-obvious logic, no magic numbers, no dead code, and consistent formatting applicable to all languages.
- Create the following twelve language-specific convention files:
  - `.aib_brain/conventions/coding-python-convention.md`
  - `.aib_brain/conventions/coding-dax-convention.md`
  - `.aib_brain/conventions/coding-sql-convention.md`
  - `.aib_brain/conventions/coding-html-convention.md`
  - `.aib_brain/conventions/coding-javascript-convention.md`
  - `.aib_brain/conventions/coding-css-convention.md`
  - `.aib_brain/conventions/coding-react-convention.md`
  - `.aib_brain/conventions/coding-flask-convention.md`
  - `.aib_brain/conventions/coding-django-convention.md`
  - `.aib_brain/conventions/coding-csharp-convention.md`
  - `.aib_brain/conventions/coding-scala-convention.md`
  - `.aib_brain/conventions/coding-uiux-convention.md`
- Modify `.aib_brain/prompts/aib-implement.md` to:
  - Unconditionally read `coding-general-convention.md` before any code generation step.
  - Conditionally read the relevant language-specific convention based on the file extension(s) being created or edited (e.g., `.py` → load `coding-python-convention.md`; `.js`/`.jsx` → load `coding-javascript-convention.md`; etc.).

## Out of scope

- Retroactive application of commenting standards to existing code files already in the repository.
- Modification of any product-doc files in `.aib_memory/docs/` as part of this request (optional documentation updates to ARCH-01 and RQT-02 are deferred to a separate request or manual edit).
- Creation of automated linting tooling or CI checks for comment coverage.
- Modifications to any AIB tool scripts in `.aib_brain/tools/`.

## Constraints

- New convention files MUST be placed exclusively inside `.aib_brain/conventions/` and MUST NOT be registered in `.aib_memory/references.md` (they are brain assets, not product-docs).
- Convention files MUST use normative language (MUST, SHOULD, MAY per BCP 14) consistent with all other AIB conventions.
- The modification to `aib-implement.md` MUST NOT remove or weaken any existing safety rules in that prompt.
- The implement step for this request requires creating files under `.aib_brain/` — the safety rule `"Do not modify .aib_brain/ assets during implementation work"` is explicitly overridden for this framework-evolution request only; this override MUST be noted in the implementation plan.

## Success criteria

- Thirteen convention files exist under `.aib_brain/conventions/` matching the pattern `coding-*.md` after implementation.
- Each convention file is non-empty (minimum 20 lines), uses normative language, and contains at least sections for: purpose, scope, comment requirements, and code quality rules.
- `.aib_brain/prompts/aib-implement.md` contains an unconditional instruction to read `coding-general-convention.md` and a conditional instruction table mapping file extensions to language-specific convention file paths.
- Running a test implement on a Python file results in generated code containing a module-level header comment and docstrings on all functions/classes.
- No existing product-doc or lifecycle files are modified as a side-effect of implementation.
