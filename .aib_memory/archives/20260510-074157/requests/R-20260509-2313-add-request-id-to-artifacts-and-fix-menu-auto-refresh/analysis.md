# Analysis — R-20260509-2313 Add request ID to artifacts and fix menu auto-refresh

---

## Executive Summary

- **Request ID:** R-20260509-2313
- **Title:** Add request ID to artifacts and fix menu auto-refresh
- **High-level purpose:** Two independent improvements to the AIB framework. First, artifact files stored at `.aib_memory/` root during an active request will be renamed to include the request ID (e.g., `request-R-20260509-2313.md`), preventing git merge conflicts when multiple development branches coexist. Second, the interactive menu's 3-second auto-refresh cycle, which stopped working after the removal of the `_REFRESH_ACTION` sentinel, will be repaired.

- **Scope summary:** Changes touch `move-request-artifacts.py`, `close-request.py`, both AIB prompts (`aib-analysis.md`, `aib-implement.md`), two conventions (`analysis-convention.md`, `request-convention.md`), `context.md`, and the test suite. The menu fix is confined to `menu.py`.

- **Request.md updates in this run:** `## Assumptions`, `## Plan`, and `## Documentation` sections were added/replaced. No Q-blocks were generated (all design decisions resolved from workspace context).

- **Risk posture:** Medium. The rename touches every AIB component that reads or writes active-request artifacts. Careful search-and-replace is required to avoid missed references. The menu fix is low-risk; the underlying TIMEOUT mechanism already exists.

- **Branch context:** This request was initiated from branch `77-move-questions-from-requestmd-to-inputmd`. The fix is directly motivated by multi-branch development friction.

## Domain Knowledge Essentials

**AIB (AI Builder):** A minimal, file-first framework for specification-driven development in a repository workspace. It manages structured work items (requests) through deterministic prompt-driven workflows.

**Active request:** A request in state `Active` in `requests_register.md`. Exactly one Active request may exist at a time (FR-001). While active, the request's primary artifact files (`request.md`, `analysis.md`, `implementation.md`, `UAT_scenarios.md`) reside at `.aib_memory/` root.

**Active-phase artifacts:** The four files stored at `.aib_memory/` root for the duration of a request: `request.md`, `analysis.md`, `implementation.md`, `UAT_scenarios.md`. After implementation, `move-request-artifacts.py` relocates them to the request subfolder (archived phase).

**Request ID:** A timestamp-based identifier generated by `create-request.py` in the format `R-YYYYMMDD-HHmi` (e.g., `R-20260509-2313`). It uniquely identifies a request without including the human-readable title slug.

**Git merge conflict:** When two branches modify or create the same file, git cannot automatically reconcile the changes and requires manual resolution. Having `request.md` on two branches causes this; having `request-R-20260509-2313.md` vs `request-R-20260510-1400.md` on two branches avoids it.

**Menu auto-refresh:** The interactive menu (`menu.py`) is designed to re-render every `_REFRESH_TIMEOUT_S = 3.0` seconds to pick up workspace state changes (e.g., when an AI agent finishes writing analysis output). This is driven by the TIMEOUT path in `choose_action()`.

**Impacted roles:**
- Developer — creates requests, runs analysis and implementation, uses the menu.
- AI Automation Agent — executes prompts; must reference the correct filenames.
- AIB Maintainer — owns `.aib_brain/` assets; must keep conventions and prompts consistent.

**Impacted business processes:** Request creation, analysis workflow, implement workflow, request closing, interactive menu navigation.

## Technical Knowledge & Terms

**`_ARTIFACT_NAMES` constant** (in `move-request-artifacts.py`): A tuple of filenames that `move_artifacts()` looks for at `.aib_memory/` root when relocating a request. Currently `("request.md", "analysis.md", "UAT_scenarios.md")`. After this request, it must become a callable or dynamic lookup using the active request ID, or it must be replaced by a pattern-based discovery.

**Two-phase placement rule:** While a request is active, artifact files live at `.aib_memory/<filename>`. After implementation completes, they are moved to `.aib_memory/requests/<request-folder>/<filename>`. This is normative per `request-convention.md` and `analysis-convention.md`.

**`msvcrt.kbhit()`:** A Windows-only function (from the `msvcrt` built-in module) that non-blockingly polls the console keyboard input buffer. Returns `True` if a key is available, `False` otherwise. Used in `menu.py`'s `get_key()` function to implement the TIMEOUT-based refresh without blocking.

**`choose_action()` loop:** The main menu event loop in `menu.py`. On each iteration, it calls `get_key(timeout=3.0)`. If no key is pressed within 3 seconds, `get_key` returns `"TIMEOUT"`, the loop `continue`s, and the menu re-renders. This is the intended auto-refresh mechanism.

**`_REFRESH_ACTION` (removed in commit 8ab54ef):** A sentinel dict that was always appended to the visible action list, ensuring `total_items >= 1`. Its removal left `total_items = 0` when no active request exists and `_SCRIPT_ACTIONS = []`. With `total_items = 0`, pressing UP or DOWN triggers a `ZeroDivisionError` (`(selected ± 1) % 0`), crashing `choose_action()` and the entire menu.

**`filter_visible_actions()`:** Returns a list of visible actions, conditionally appending the Close Request action when an active request exists. With `_SCRIPT_ACTIONS = []` and no active request, returns `[]`.

**ConPTY (Windows Console Pseudoconsole):** Microsoft's PTY implementation used by VS Code's integrated terminal on Windows. Provides a real Windows console handle, meaning `msvcrt.kbhit()` should work correctly within VS Code's terminal.

**Non-functional attributes:**
- Reliability: The rename must be atomic per artifact; partial renames leave the system in an inconsistent state.
- Maintainability: All artifact path references must be centralized (ideally in `common.py`) rather than scattered across prompts and tools.
- Testability: Tests must be updated to use the ID-suffixed filenames.

**Findings:**

1. **Artifact file references are widespread.** The strings `request.md`, `analysis.md`, `implementation.md`, `UAT_scenarios.md` appear in: both prompts, both conventions, `move-request-artifacts.py`, `context.md`, `Concepts.md`, and multiple test files. A search-and-replace approach is necessary; the risk of a missed reference is high.

2. **Root cause of auto-refresh failure identified.** Commit `8ab54ef` removed `_REFRESH_ACTION` from the action list. Previously, `script_actions = filter_visible_actions(...) + [_REFRESH_ACTION]` guaranteed `total_items >= 1`. Now `total_items = 0` when no active request exists, causing `ZeroDivisionError` on UP/DOWN keypress. The TIMEOUT path itself (line `if key == "TIMEOUT": continue`) is intact and correct. The fix is a two-line guard: `if total_items > 0:` before each modulo operation.

3. **`_ARTIFACT_NAMES` is a static tuple.** Currently `("request.md", "analysis.md", "UAT_scenarios.md")`. To support ID-suffixed names, `move_artifacts()` needs to know the active request ID. It already reads the register to find the folder, so extracting `request_id` there is straightforward.

4. **`implementation.md` is excluded from `_ARTIFACT_NAMES`.** Despite being an active-phase artifact, `implementation.md` is NOT in `_ARTIFACT_NAMES`. This is intentional per `aib-implement.md` — `implementation.md` is created directly in the request subfolder, not at `.aib_memory/` root. This means it does NOT need to be renamed using the ID scheme at the root level; it already resides in its final location. Only `request.md`, `analysis.md`, and `UAT_scenarios.md` need renaming at the root level.

5. **Convention files are normative.** `analysis-convention.md` specifies `analysis.md` as the exact filename. `request-convention.md` specifies `request.md`. Both must be updated to reflect the new naming convention, which is a non-trivial normative change.

6. **`aib-implement.md` reads `request.md` from `.aib_memory/request.md`.** This is explicitly stated: "Read it from `.aib_memory/request.md`". This path reference must be updated to `request-<request_id>.md`.

7. **Tests heavily reference bare filenames.** `test_artifact_placement.py` creates bare `request.md`, `analysis.md`, `UAT_scenarios.md` files and asserts their movement. These must be updated. `test_menu.py` is unaffected by the rename (it doesn't reference artifact filenames). Other test files may reference these names.

## External Benchmarking

**Branch-scoped artifact naming (Git workflow patterns):**
- A well-established pattern in multi-branch development is to embed a discriminator in artifact filenames to prevent merge conflicts. For example, database migration frameworks (Alembic, Flyway, Django migrations) use timestamp or sequence prefixes on migration files — each branch produces a uniquely named file that doesn't conflict during merge. AIB's adoption of `request-<ID>.md` directly mirrors this pattern.
- Key takeaway: The timestamp-based request ID (`R-YYYYMMDD-HHmi`) is already well-suited as a discriminator because it is unique per request and does not include human-readable slugs that might change.
- Applicability: High. The approach is proven, minimal, and doesn't require any structural changes to the repository layout.

**Terminal polling for responsive CLI refresh (Curses / PTY patterns):**
- Libraries like `curses` (Unix) and `windows-curses` use non-blocking poll loops with short sleep intervals to achieve responsive terminal UIs. The pattern `while not kbhit(): sleep(0.05); if timeout_reached: return TIMEOUT` used in `menu.py` is a standard implementation. The root cause of the AIB menu failure is not in this pattern but in a missing zero-division guard — a common oversight when transitioning from "always at least one item" to "potentially zero items."
- Key takeaway: Guard all modulo operations that use a list length as denominator against the zero case; this is a standard defensive programming practice.
- Applicability: The fix is already identified: add `if total_items > 0:` guards before `selected = (selected ± 1) % total_items`.

## Minimal Spikes and Experiments

**Spike: Confirm root cause of menu ZeroDivisionError**
- Hypothesis: `choose_action()` crashes with `ZeroDivisionError` when `total_items = 0` and the user presses UP or DOWN.
- Approach: Static code review of `choose_action()` in `menu.py` post-commit `8ab54ef` combined with inspection of the git diff removing `_REFRESH_ACTION`.
- Outcome: Confirmed. Lines `selected = (selected - 1) % total_items` and `selected = (selected + 1) % total_items` raise `ZeroDivisionError` when `total_items = 0`. The TIMEOUT path (`if key == "TIMEOUT": continue`) is unaffected and works correctly.
- Conclusion: The fix is a two-line guard. The auto-refresh TIMEOUT mechanism itself is correct and does not need to be rewritten.

**Spike: Confirm `implementation.md` placement in the active phase**
- Hypothesis: `implementation.md` is created inside the request subfolder (not at `.aib_memory/` root) during the active phase, so it does not need to be ID-suffixed at the root.
- Approach: Read `aib-implement.md` instructions; confirm `implementation.md` is NOT listed in `_ARTIFACT_NAMES` in `move-request-artifacts.py`.
- Outcome: Confirmed. `aib-implement.md` creates `implementation.md` directly in the active request folder. `move-request-artifacts.py` does not include it in `_ARTIFACT_NAMES`. No root-level `implementation.md` is written.
- Conclusion: Only `request.md`, `analysis.md`, and `UAT_scenarios.md` need ID-suffixed renaming at `.aib_memory/` root. `implementation.md` is out of scope for the rename.

## AI Copilot Suggestions

**Observation 1 — Centralize artifact filename resolution in `common.py` (maintainability risk)**
The artifact filenames (`request.md`, `analysis.md`, `UAT_scenarios.md`) are currently referenced as string literals scattered across two prompts, two conventions, one tool script, and tests. After this change, the ID-suffixed names will also be referenced in multiple places. If the naming convention changes again, every reference must be updated individually. A better long-term design is to add a helper function in `common.py` (e.g., `artifact_name(artifact_type, request_id)`) that returns the correct filename, and have all Python tools call it. However, prompts (plain Markdown) cannot call Python functions — they must use string patterns.
- Suggestion: For Python tools and tests, centralize filename construction in `common.py`. For prompts and conventions, document the pattern clearly and update in one place (the convention file that both prompts import by reference).

**Observation 2 — Scope is larger than strictly necessary for the stated goal (scope risk)**
The developer's goal is to avoid merge conflicts. A minimal fix would rename only the files that cause conflicts. However, `implementation.md` is already in the request subfolder (no conflict risk), and `UAT_scenarios.md` is optional. The prompt and convention updates add significant scope. The scope as defined is correct and necessary for consistency, but the implementer should be aware that a partial implementation (e.g., renaming only `request.md` and `analysis.md`) would achieve the core goal with less risk.
- Suggestion: Implement the full scope as defined, but tackle `move-request-artifacts.py` and the prompt references first (highest conflict risk), then conventions and context.md, then tests.

**Observation 3 — The menu ZeroDivisionError is a regression introduced by `8ab54ef` (implementation risk)**
The auto-refresh failure was reported vaguely ("not working"), but the root cause is a crash triggered by UP/DOWN keypresses when no actions are visible. This means the menu appears to work (renders correctly, refreshes via TIMEOUT) until the user navigates with arrow keys, at which point it crashes silently. The developer may not have experienced the crash if they only used digit keys or 'q'.
- Suggestion: The guard `if total_items > 0:` before each modulo is the minimal correct fix. Additionally, a test case for `choose_action()` with `total_items = 0` should be added to prevent this regression from recurring.

**Observation 4 — Convention files are normative and normative changes are high-risk**
`analysis-convention.md` and `request-convention.md` are read by `aib-analysis.md` on every run. Any error in the updated convention (e.g., an ambiguous pattern description) will silently break future analysis runs. These files must be updated with precision, and the updated pattern must be unambiguous (e.g., exactly `<artifact-type>-<request_id>.md`).
- Suggestion: Test the updated conventions immediately after editing by running `pytest tests/` and a dry-run of the analysis prompt on a scratch request.

## Multi-Perspective Stakeholder Review

### Senior Solution Architect

The request correctly identifies a real architectural weakness: active-phase artifacts using fixed names in a shared directory is incompatible with parallel-branch development. The proposed naming scheme (`<artifact-type>-<request_id>.md`) is minimal and deterministic. However, the implementation surface is wide — references exist in prompts (plain text), conventions (normative text), Python scripts, and tests. A partial implementation (updating some references but not others) will produce a broken system. The architect's primary concern is consistency: once the rename is in place, every component that reads or writes an active artifact must use the new name. The absence of a centralized `artifact_name()` helper in `common.py` is the main long-term design risk. For this iteration, the scope is proportionate to the goal.

- The two-phase placement rule (active at root, archived in subfolder) is sound and should be preserved.
- Consider introducing an `artifact_name(artifact_type, request_id)` helper in `common.py` to centralize construction.
- The archived-phase filenames (inside the request subfolder) do not need to change — they are already disambiguated by folder.
- Conventions must be updated atomically with the tool changes to prevent a window where the convention and the tool disagree.

### Product Owner

This request delivers two user-facing improvements: a quality-of-life fix (menu refresh) and a workflow enabler (multi-branch development). Both are clearly articulated and have measurable success criteria. The scope is appropriately bounded — it touches infrastructure without changing the user-facing workflow. The request ID format is already established, so no new user-facing concept is introduced. Risk to existing functionality is the main concern: any regression in the analysis or implement flow would block the primary developer workflow.

- SC-07 (pytest passes) is the key acceptance gate.
- The request would benefit from a UAT scenario covering an actual two-branch scenario, though this is difficult to automate.
- Value is medium-high: the menu fix unblocks daily frustration; the rename unblocks a future workflow.

### User

From the developer's perspective, the change is transparent: the artifact filenames become slightly longer but follow a predictable pattern. The menu "just works" after the fix. The main friction point is that existing active requests (during the migration window) use bare filenames — the developer must be aware that a request created before this change uses the old naming convention and `move-request-artifacts.py` will fail to find ID-suffixed files if it is updated without handling the migration. This is the one scenario where user confusion is likely.

- Clear documentation of the new naming pattern in the convention files is essential.
- No new developer action is required after the change — the filenames are generated by tools, not typed manually.
- The 3-second auto-refresh is the expected behavior; restoring it reduces cognitive load during long AI agent runs.

### Security Officer

No new security surface is introduced. Artifact filenames are derived from request IDs that are timestamp-based and generated by trusted tooling (`create-request.py`). No user-supplied input is directly interpolated into filenames without sanitization (the request ID is already validated as `R-YYYYMMDD-HHmi` format). The menu fix does not introduce any network calls or privilege escalation. All operations remain local to the workspace.

- Ensure the `artifact_name()` helper (if introduced) validates or sanitizes the `request_id` parameter to prevent path traversal (e.g., `request_id` containing `../`).
- No auth/authz impact.

### Data Governance Officer

All artifacts are internal engineering documentation with no PII, regulated data, or external data dependencies. The rename changes only filenames, not content or retention rules. Archived artifacts in request subfolders remain unchanged. The `requests_register.md` remains the authoritative audit trail for request lifecycle. No lineage, classification, or compliance impact is identified.

- Ensure that any backup or CI artifact collection logic that references bare filenames (e.g., artifact upload steps in GitHub Actions) is reviewed. No such steps were identified in the current workspace.
- The convention update is a normative change and should be captured in the next version log.
