## Goal

Restructure `aib-analyze.md` for a clearer linear execution sequence, separate structural content into `analysis-convention.md`, update Q-block format to support free-text questions and multiple options, rename "Implementation Alternatives" to "Decision Register" with enriched resolution semantics, and remove several obsolete restrictions.

## Constraints

- Maintain backward compatibility with the existing Answer Application Sub-flow logic (section 7.2).
- The prompt must remain model-agnostic (NFR-001).
- Convention changes must not break the mandatory section list (analysis documents must still have 5 required sections).
- Tool scripts (`finalize-input.py`, `create-request.py`) must not be modified.
- Follow AIB convention-over-configuration principle.

## Success criteria

- The analyze prompt follows the 9-step linear sequence described in the input.
- Section 3.3 (Context-Window Management) is absent from the prompt.
- All structural/formatting content (heading definitions, content schemas, sub-section structure) resides only in `analysis-convention.md`, not duplicated in the prompt.
- The "Do NOT embed external links" instruction is absent from both the prompt and the convention.
- The "Soft limit: 9 Q-blocks per run" line is absent from the prompt.
- Q-block format includes a free-text question variant (no options, only `- Answer: ___`).
- Q-block format removes `> Answer:` line.
- Q-block format example shows more than 2 options.
- `analysis-convention.md` section 4.5 is renamed to "Decision Register" with the specified structure.
- `analysis-convention.md` section 5 (Maintenance Rules) is removed.
- `analysis-convention.md` section 7 no longer contains "Given the same memory state and request input, analysis output intent must be identical."
- `analysis-convention.md` section 8 (Prohibited Content) no longer lists "External hyperlinks" or "Embedded images or diagrams".
- All existing tests pass after changes.
