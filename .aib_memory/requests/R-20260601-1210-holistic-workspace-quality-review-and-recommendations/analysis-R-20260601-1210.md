### Industry Knowledge

1. **Prompt Engineering Best Practices (OpenAI, Anthropic documentation):**
   - Prompts should minimise internal cross-referencing to reduce hallucination risk. The current `aib-analyze.md` (400+ lines) has high cognitive load with 11 steps, multiple sub-steps, appendices, and internal variable tracking. Industry guidance recommends modular prompts with clear single-responsibility where possible.
   - *Applicability:* Moderate. The single-prompt monolithic approach works for determinism but trades off against maintainability and LLM context-window pressure in smaller models.

2. **Convention-over-Configuration (Ruby on Rails, Spring Boot patterns):**
   - Successful convention-driven frameworks keep conventions discoverable and minimal. AIB's 20 convention files represent a comprehensive but potentially overwhelming rule set. Industry best practice suggests progressive disclosure — core conventions mandatory, extended ones opt-in.
   - *Applicability:* High. New users face a steep learning curve with 20 convention files. Consider a "convention index" or tiered importance levels.

3. **Git Repository Size Management (GitHub best practices):**
   - Storing binary archives (zip files) in git repositories inflates clone times and is discouraged. GitHub recommends using Releases/Artifacts or Git LFS for versioned binaries. The current `versions/` folder with 30 zip files will grow unboundedly.
   - *Applicability:* High. Migration to GitHub Releases or LFS would significantly reduce repository bloat and improve clone performance.
