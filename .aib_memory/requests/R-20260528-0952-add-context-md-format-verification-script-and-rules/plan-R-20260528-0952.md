## Goal

Create a Python verification script (`verify-context.py`) that validates `.aib_memory/context.md` against the context convention, amend the context convention to replace stub notices with atomic informational statements, update existing stubs in `context.md`, add a dedicated context review step in `aib-analyze.md`, and add verification invocation instructions to prompts that modify `context.md`.

## Constraints

- Python 3 standard library only (no third-party dependencies).
- Convention (`context-convention.md`) is the source of truth for all checks.
- `.aib_brain/` modifications are explicitly authorized by this request.
- Separation of concerns: `verify-context.py` is a standalone tool; calling prompts invoke it after modifications.
- The verification script reports only; it does not auto-correct.
- Exit code 0 means all checks pass; exit code 1 means one or more checks fail.

## Success criteria

- `verify-context.py` exists at `.aib_brain/tools/verify-context.py` and executes without error on the current `context.md`.
- Running `python .aib_brain/tools/verify-context.py --workspace .` exits with code 0 on a well-formed `context.md`.
- Running the script on a malformed `context.md` (missing section, duplicate index, invalid statement format) exits with code 1 and prints structured pass/fail output.
- `context-convention.md` no longer contains the `*Not yet documented.*` stub format; it requires atomic informational statements instead.
- `context.md` subsections `### 2.13 Analytics - AN` and `### 2.16 Performance - PF` contain atomic informational statements instead of `*Not yet documented.*`.
- `aib-analyze.md` contains an 11-step workflow with the new context review step as S06 and subsequent steps renumbered S07-S11.
- A test file `tests/test_verify_context.py` exists and all tests pass.
- `python -m pytest tests/test_verify_context.py` exits with code 0.

## Plan

### Task 1: Create verify-context.py

#### Intent

Implement the format verification script for `context.md`.

#### Outputs

`.aib_brain/tools/verify-context.py`

#### Procedure

Create `.aib_brain/tools/verify-context.py` with the following structure:

- Accept `--workspace` argument (default `.`) via argparse.
- Read `.aib_memory/context.md` from the workspace.
- Implement individual check functions, each returning (pass: bool, message: str):
  1. `check_preamble` — Verify document starts with `# Product Context` and the auto-generated blockquote preamble.
  2. `check_section_structure` — Verify exactly three H2 sections exist in order: `## 1. Product Identity`, `## 2. Statements`, `## 3. Workspace File Inventory`.
  3. `check_subsection_headings` — Verify all 22 subsection headings exist in the correct order under `## 2. Statements`.
  4. `check_subsection_non_empty` — Verify every subsection has at least one atomic statement line (no empty subsections).
  5. `check_statement_format` — Verify every bullet line in Section 2 matches the pattern `- <AREA>-<TYPE>-<HASH>: <text>` with valid area prefix, type letter, and 8-char hex hash.
  6. `check_statement_uniqueness` — Verify no duplicate `<AREA>-<TYPE>-<HASH>` indices exist.
  7. `check_no_external_hyperlinks` — Verify no `http://` or `https://` strings appear anywhere in the file.
  8. `check_no_html_tags` — Verify no HTML tags appear in the file.
  9. `check_no_tables` — Verify no Markdown table syntax (`|`) appears in the file.
  10. `check_inventory_present` — Verify `## 3. Workspace File Inventory` has at least one bullet entry.
  11. `check_product_identity_non_empty` — Verify `## 1. Product Identity` has substantive prose content.

- Run all checks and print results in format: `[OK] <check_name>` or `[FAIL] <check_name>: <message>`.
- Print summary line: `Results: <passed>/<total> checks passed.`
- Exit with code 0 if all pass, 1 if any fail.

#### Done criteria

- File exists at `.aib_brain/tools/verify-context.py`.
- Script runs without import errors.
- Produces structured output for both passing and failing cases.

#### Dependencies

None.

#### Risk notes

Not all 12 quality gates from the convention are programmatically verifiable (e.g., "rebuildability" and "determinism" require semantic judgment). The script implements the structurally verifiable subset.

### Task 2: Amend context-convention.md stub format

#### Intent

Replace the `*Not yet documented.*` stub notice format with a requirement for atomic informational statements.

#### Outputs

`.aib_brain/conventions/context-convention.md`

#### Procedure

Edit `.aib_brain/conventions/context-convention.md`:

Replace the "Stub Notice Format (Normative)" section content. Remove the requirement for `*Not yet documented.*`. Replace with: "When a mandatory subsection has no applicable content or no information has been found yet, the section body MUST contain at least one atomic informational statement explaining the situation. Use format `<AREA>-I-<HASH>: <explanation>.` where `<explanation>` states why the area is not applicable or that no information has been found yet."

Update quality gate #3 ("Stub sections present") to reference the new atomic informational statement requirement instead of the old stub notice format.

#### Done criteria

- No reference to `*Not yet documented.*` as a required format exists in `context-convention.md`.
- The new atomic informational statement requirement is clearly stated.
- Quality gate #3 references the new format.

#### Dependencies

None.

#### Risk notes

The `aib-refresh-context.md` prompt currently generates `*Not yet documented.*` stubs. It reads the convention for guidance, so it should automatically adapt. Verify during testing.
