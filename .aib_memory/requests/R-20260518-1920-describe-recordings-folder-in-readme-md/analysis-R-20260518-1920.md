### Industry Knowledge

1. **Progressive disclosure in documentation (Write the Docs / Divio documentation system):** Best practice distinguishes between "tutorial" content (step-by-step, learning-oriented) and "reference" content (exhaustive, lookup-oriented). A root README is primarily a landing page and orientation guide. The recordings section fits the "tutorial" category and should be brief in the README — a short paragraph and a list of videos suffices, with the actual content residing in the folder. Applicability: high — confirms the README should reference the folder without embedding video content or lengthy descriptions.

2. **Discoverability-first README structure (GitHub documentation guidelines):** GitHub recommends that a README be the first document a visitor reads. Resources like video tutorials should appear early enough that they are not missed but after essential onboarding information (overview, installation). Applicability: high — placement after the Installation section optimises discoverability for users who have just completed setup and are looking for next steps.

3. **Sequential naming conventions for tutorials (Google Developer Documentation Style Guide):** Numbered file names (01_, 02_, ...) imply a recommended viewing order. Documentation best practice is to surface this ordering explicitly to users rather than relying on alphabetical sort order in file explorers. Listing the videos with a brief descriptor per entry is more informative than a generic "see the recordings/ folder" mention. Applicability: high — the existing file naming already follows this pattern; the README description should reflect the sequential structure.
