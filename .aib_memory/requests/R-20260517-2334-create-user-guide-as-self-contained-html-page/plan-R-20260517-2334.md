## Goal

Create a single self-contained HTML file at `.aib_brain/user_guide.html` that serves as a browser-viewable user guide for the AI Builder (AIB) framework. The file must embed all CSS styling and JavaScript behavior inline (no external CDN or network dependency at runtime) and cover the full AIB usage surface: workspace initialization, the daily analyze → implement workflow, prompt invocations, the Q&A cycle, `instructions.md` directives, workspace folder structure, and the AIB concepts/glossary. Navigation uses an accordion pattern — all section headers are always visible; clicking a header expands or collapses that section's content.

## Constraints

- Output is a single file: `.aib_brain/user_guide.html`. Writing to `.aib_brain/` is authorized by the active request scope.
- All CSS and JavaScript MUST be embedded inline in the HTML file; no `<link>` to external stylesheets, no `<script src="...">` referencing external scripts, no CDN references at runtime.
- Navigation MUST use the accordion pattern: all section headers always visible; clicking a header toggles expand/collapse of that section's content. Resolved from Q001 answer: Option C — Accordion sections.
- Icons MUST be sourced from an open-source, MIT-licensed library (e.g., Heroicons, Feather Icons, Tabler Icons) and embedded as inline SVG within the HTML markup; no binary media files required unless inline embedding is not feasible.
- The optional `.aib_brain/media/` folder is NOT created for this request; all assets are inline.
- No modifications to existing `.aib_brain/` prompts, conventions, or tool scripts are permitted.
- No changes to `.aib_memory/` files beyond `context.md`, and no changes to workspace source code, tests, or CI workflows.
- The guide is a purely static artifact; no backend, server-side rendering, or build toolchain is required.
- Python 3.10+ standard library only; no new Python dependencies introduced.

## Success criteria

- File `.aib_brain/user_guide.html` exists and can be opened in a standard browser without network connectivity.
- The file contains no `<link>` to external stylesheets and no `<script src="...">` referencing external JavaScript files; all CDN references are absent.
- Accordion navigation is functional: each section header is always visible; clicking a header expands its content; clicking again collapses it.
- Content covers all workflow sections documented in `.aib_brain/README.md`: Introduction, Prerequisites, Quick Start, Daily Flow, Prompt Invocations, Use Cases, Q&A Mechanism, Workspace Instructions, Folder Structure, Concepts/Glossary, and Request Folder Artifacts.
- Icons used (if any) originate from an open-source MIT-licensed library and are embedded as inline SVG.
- `.aib_memory/context.md` reflects the existence of `user_guide.html` and all stale section-name references ("Assumptions", "Documentation" as plan sections) are corrected.
- `.aib_brain/README.md` no longer uses the stale plain `analysis.md` filename without the request-ID suffix.

## Plan

### Task 1: Create .aib_brain/user_guide.html

#### Intent
Create the self-contained HTML user guide file with accordion navigation, inline CSS/JS, and complete content coverage of all AIB workflow sections.

#### Outputs
`.aib_brain/user_guide.html` (new file)

#### Procedure
Read `.aib_brain/README.md` and `.aib_memory/context.md` (Glossary section) to map all content that must appear in the guide before authoring any HTML.

Create `.aib_brain/user_guide.html` as a new file with the following top-level structure: `<!DOCTYPE html>`, `<html lang="en">`, `<head>` containing `<meta charset="UTF-8">`, `<meta name="viewport" ...>`, `<title>AIB User Guide</title>`, and a `<style>` block. The `<body>` contains the page header, all accordion sections, and a `<script>` block as the last child of `<body>`.

In the `<style>` block, define: base page styles (font, color scheme, max-width container), `.accordion-item` wrapper, `.accordion-header` (cursor: pointer, flex layout, border, padding, hover state), `.accordion-content` (display: none; padding; overflow: hidden), `.accordion-content.open` (display: block), and code/pre styles for inline command display.

In the `<script>` block, implement accordion toggle logic: query all `.accordion-header` elements, attach a `click` event listener to each that toggles the `.open` class on the adjacent `.accordion-content` sibling element (~30–40 lines of plain JavaScript, no framework).

Populate the `<body>` with the following accordion panels in order — each as `<div class="accordion-item">` with a `<button class="accordion-header">` and `<div class="accordion-content">`:
(1) Introduction — AIB overview and objectives drawn from `.aib_brain/README.md` Objectives section;
(2) Prerequisites — Python version, terminal requirement from `.aib_brain/README.md` Prerequisites;
(3) Quick Start — `run.bat` / `run.sh` invocation and menu description from `.aib_brain/README.md` Quick Start;
(4) Daily Flow — 3-step daily flow from `.aib_brain/README.md` Daily Flow;
(5) Prompt Invocations — `aib-analyze.md`, `aib-implement.md`, `aib-refresh-context.md` invocation syntax from `.aib_brain/README.md` Prompt Invocations;
(6) Use Cases — three use case patterns (full analyze→implement, direct implement, refresh context) from `.aib_brain/README.md` Use Cases;
(7) Questions and Answers — full Q&A cycle description from `.aib_brain/README.md` Questions and Answers;
(8) Workspace Instructions — `instructions.md` purpose and usage from `.aib_brain/README.md` Workspace Instructions;
(9) Folder Structure — folder tree from `.aib_brain/README.md` Folder structure section;
(10) Request Folder Artifacts — artifacts table from `.aib_brain/README.md` Request Folder Artifacts;
(11) Concepts/Glossary — all glossary terms from `.aib_memory/context.md` Glossary section.

Embed at least one inline SVG icon (e.g., a chevron-down/up indicator) in the accordion header buttons sourced from Heroicons (MIT) to indicate expand/collapse state. The SVG `transform` or `class` should toggle on open state via CSS (e.g., `rotate(180deg)` for chevrons).

#### Done criteria
- File `.aib_brain/user_guide.html` exists.
- No `<link rel="stylesheet">` or `<script src="...">` referencing external URLs in the file.
- All 11 accordion sections are present.
- `<style>` block contains `.accordion-header` and `.accordion-content` rules.
- `<script>` block contains click event listener logic for accordion toggle.

#### Dependencies
None

#### Risk notes
Heroicons SVG markup is MIT-licensed; inline SVG use requires no attribution. Verify the icon markup is valid HTML before writing the file. Accordion interactive behavior can only be verified by opening the file in a browser — this is outside automated test scope.
