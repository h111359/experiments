## Goal

Edit `.aib_memory/context.md` to split every bullet item that exceeds two sentences into two or more shorter bullets, each containing at most two sentences, with all original content preserved verbatim and no paraphrasing permitted. Additionally, add a mandatory Rule 16 verification step to `.aib_brain/prompts/aib-context.md` so that future regenerations of `context.md` cannot persist bullet items exceeding two sentences without explicit correction before write.

## Constraints

- Only `.aib_memory/context.md` and `.aib_brain/prompts/aib-context.md` may be modified in this request.
- Each resulting bullet must be a grammatically complete sentence ending with a period (Rule 14 of `context-convention.md`).
- One blank line must separate every top-level list item after all splits (Rule 12 of `context-convention.md`).
- Content of existing bullets must not be summarized, paraphrased, or altered; structural splitting only.
- Changes to `aib-context.md` must not modify existing phase logic beyond inserting the Rule 16 verification step.
- Additional out-of-scope discrepancies identified during analysis audit (duplicate Workspace File Inventory entry, stale artifact path names, malformed bullet, missing path prefix) must NOT be fixed in this request.

## Success criteria

1. Every bullet item in `.aib_memory/context.md` contains at most two sentences (Rule 16 satisfied).
2. One blank line separates every top-level list item throughout the document (Rule 12 preserved).
3. Every bullet is a grammatically complete sentence ending with a period (Rule 14 preserved).
4. `pytest tests/test_context_formatting_rules.py` passes with exit code 0.
5. No content is lost or paraphrased; a diff shows only additions of blank lines and bullet markers for newly split items.
6. `.aib_brain/prompts/aib-context.md` contains an explicit, named Rule 16 verification step positioned before the file write action in Phase 5.
