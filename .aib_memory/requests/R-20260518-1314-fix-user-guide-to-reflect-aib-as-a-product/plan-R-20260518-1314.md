## Goal

Correct all content in `.aib_brain/user_guide.html` that conflates the AI_Builder development project with AIB as a general product. Six specific errors have been identified; each must be corrected so that the user guide accurately describes what AIB users need and see when installing and running AIB in any workspace, not just the AI_Builder repository.

## Constraints

- `.aib_brain/user_guide.html` must remain a valid, self-contained HTML file with no external dependencies after editing.
- Changes are limited to content text; the accordion structure, CSS, JavaScript, and section layout must not be altered.
- No new sections may be added to the user guide.
- `user_guide.html` is in `.aib_brain/` which is normally off-limits to tool script writes; edits are performed by the AI agent directly (not via tool scripts).
- The six identified errors are the only required changes; no speculative improvements outside the identified scope are permitted.

## Success criteria

- SC-1: Section 2 of `user_guide.html` does not contain any mention of Git or PATH as a user prerequisite.
- SC-2: The `.aib_brain/` folder tree in Section 9 lists `user_guide.html` with an accurate description.
- SC-3: The `README.md` annotation in the `.aib_brain/` folder tree in Section 9 no longer reads "This guide's source of truth"; it describes README.md's actual role.
- SC-4: The `.aib_memory/logs/` folder in the Section 9 folder tree is described as tool execution and action logs, not version logs or curated change bullets.
- SC-5: The CI glossary entry in Section 11 does not contain a branch-specific reference (`main`).
- SC-6: The VCS glossary entry in Section 11 does not contain the phrase "in this workspace".
- SC-7: All existing automated tests pass without modification after the changes are applied.
