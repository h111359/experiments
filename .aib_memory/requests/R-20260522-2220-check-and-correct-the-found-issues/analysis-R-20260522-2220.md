## Research Results

### Workspace Pattern-Scan Findings

- `analysis-convention.md` section 6 (Determinism Rules) contains the bullet: "create a `Q<nnn>` question block in `plan-<request_id>.md` -> `## Decisions` instead of making assumptions." The authoritative source (`aib-analyze.md` section 6.3.3) specifies Q-blocks are written to `input.md ## Questions`. The `plan.md ## Decisions` section records resolved Q&A entries (append-only), not open Q-blocks. Contradiction confirmed.

- `analysis-convention.md` section 2 lists `Location: .aib_memory/requests/<request-folder>/` — this describes only the archived-phase placement. The active-phase placement (`.aib_memory/analysis-<request_id>.md` at the `.aib_memory/` root) is absent. Section 3 of the same file correctly documents the two-phase placement rule, creating an internal inconsistency between sections 2 and 3.

- `analysis-convention.md` section 2 also lists `Target files: analysis.md` — this omits the `<request_id>` suffix that is mandatory per section 3's naming rule. This is an additional inconsistency not in the current request scope but noted for awareness.

- `aib-analyze.md` contains two near-identical paragraphs: section 5.7 (Step 7 — Quality Check) and section 6.1 (Analysis Document output spec). Both instruct the agent to evaluate `requirements-analysis-convention.md` items and surface a Requirements Gate Evaluation sub-heading in `## Research Results`. The section 6.1 paragraph differs from 5.7 only in minor wording ("After reading..." vs. "After generating..."; section 6.1 omits "in step 8" for Q-block generation). This creates ambiguity about whether the quality check runs twice.

- `write_analysis.py` contains: `pathlib.Path(r'c:\Hristo\projects\AI_Builder\analysis_new.md').read_text(...)` and `pathlib.Path(r'c:\Hristo\projects\AI_Builder\.aib_memory\analysis.md').write_text(...)`. Both paths are hardcoded to a non-existent absolute path (`c:\Hristo\projects\AI_Builder\`), and the target uses the obsolete `analysis.md` naming without a request-ID suffix. No test or prompt references this file as a dependency. The script would fail on execution.

- `.aib_brain/README.md` line 4: "AI Builder (AIB) is a minimal but powerful framework for AI specification based development called." — the word "called" is a dangling fragment with no referent, making the sentence grammatically incomplete.

- `aib-refresh-context.md` non-goals: "Do not explore or read `.aib_brain/` folder contents except `.aib_brain/conventions/context-convention.md`." The same prompt's Reverse-Engineering Evidence Collection section instructs: "Use `.aib_brain/tools/file-inventory.py` to emit a JSONL inventory and compare with the existing list in `.aib_memory/context.md`." This creates a logical tension since `file-inventory.py` is inside `.aib_brain/tools/`, which is nominally excluded by the non-goals rule.

- `.aib_memory/instructions.md` README maintenance directive uses backslash separators: `.aib_brain\prompts\aib-implement.md`, `.aib_brain\README.md`, `.aib_memory\context.md`. All other AIB documentation uses forward slashes.

### Industry Knowledge

- **Documentation consistency principle** (IEEE 830 / IEEE 29148): Normative documents must be internally consistent; when a rule appears in multiple places, one must be designated authoritative and others must cross-reference it. A cross-reference in section 6.1 of `aib-analyze.md` pointing to section 5.7 satisfies this principle for F-006.

- **Dead code elimination** (Martin Fowler's Refactoring, Chapter 8): Unused or broken utility scripts increase cognitive load and create false expectations about workspace capabilities. Removal is preferred over update when no active consumer exists, which directly justifies Option A for F-005.

- **Path separator portability** (POSIX.1 / Git conventions): Forward slashes are universally supported across operating systems in path references within documentation and configuration files. Git itself normalizes to forward slashes. This supports F-014's correction.

- **Principle of least surprise** (API design and documentation best practices): When a rule lists exceptions, all exceptions must be enumerated. A non-goals clause that silently permits tool invocations contradicts the explicit prohibition without warning, violating the principle of least surprise. The F-008 fix restores this by making the exception explicit.
