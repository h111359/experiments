# Analysis: R-20260523-1702

## Overview

**Request ID:** R-20260523-1702  
**Request title:** Generate q-block-convention.md from aib-analyze.md

### Background

`aib-analyze.md` currently co-locates Q-block format definitions (the exact markdown templates, QID format, `> **Why this matters:**` requirement, `*(recommended)*` marker rules) alongside behavioral rules (process steps, classification rules, generation conditions, and format selection guidance). Convention files are the authoritative location for structural and formatting definitions across the AIB framework. Having format template definitions inside a prompt file violates this separation-of-concerns principle and makes format maintenance harder: any Q-block appearance change requires editing a behavior-defining prompt rather than a structure-defining convention. Format selection guidance (the logic governing when to choose multiple-choice versus free-text) is behavioral in nature and belongs in the prompt.

### Scope

- Create `.aib_brain/conventions/q-block-convention.md` as the authoritative source for Q-block format templates and format-level rules.
- Extract from `aib-analyze.md` section 6.3: both Q-block format templates (multiple-choice and free-text), their format-level rules (QID numbering, `> **Why this matters:**` requirement, `*(recommended)*` marker rules, option checkbox format, `- Answer:` format, free-text content requirement).
- Format selection guidance ("multiple-choice preferred when bounded options exist" and "use free-text only when the answer space is unbounded") stays in `aib-analyze.md` as behavioral decision logic.
- Update `aib-analyze.md` section 6.3 to remove extracted format template content and reference `q-block-convention.md` for format specification.
- Update `tests/test_questions_in_input_md.py`: redirect the two tests that assert Q-block format strings from `aib-analyze.md` to `q-block-convention.md`.
- Update `.aib_memory/context.md` to acknowledge the new convention file.

### Out of scope

- No behavioral changes to Q-block classification, generation triggers, or the Answer Application Sub-flow.
- No changes to sections 6.1 (Decision Identification) or 6.2 (Decision Classification) in `aib-analyze.md`.
- No changes to format selection guidance in `aib-analyze.md` section 6.3 (this stays in the prompt).
- No changes to `input.md` seed templates, `finalize-input.py`, `create-request.py`, or any other tool scripts.
- No changes to other prompts (`aib-implement.md`, `aib-refresh-context.md`, etc.).
- No changes to `analysis-convention.md`, `plan-convention.md`, or `requirements-analysis-convention.md`.

### Industry Knowledge

- **Separation of concerns (Robert C. Martin, Clean Code; Gang of Four principles):** Storing format/structure specifications in dedicated convention files separate from executable behavior prompts is a direct application of the single-responsibility principle. A convention file defines shape; a prompt defines process. Mixing both in a single file couples two different types of change, increasing maintenance risk. Applicability: directly supports the rationale for extracting format templates to a convention file.

- **Format specification vs. behavioral logic distinction (IETF RFCs; documentation engineering practice):** RFC 2119-style convention documents routinely separate "what the artifact looks like" from "when and how to use it." The former is structural (belongs in format spec / convention files); the latter is procedural (belongs in executable guides, prompts, or runbooks). The developer's correction precisely aligns with this principle: format selection guidance is procedural ("when to use which format") and belongs in the prompt. Applicability: validates the corrected DP-3 resolution.

- **Convention over configuration (Ruby on Rails community; widely adopted in software frameworks):** When a framework provides authoritative convention files for structural definitions, practitioners benefit from a single, predictable location for format information. This reduces cognitive overhead during Q-block authoring — the practitioner consults `q-block-convention.md` for the template, then `aib-analyze.md` for when and where to apply it. Applicability: confirms the value of the new convention file as the dedicated format reference.

### Decision Points

#### Choice: Which content from section 6.3 moves to the convention vs. stays in the prompt?

- **Tag:** `resolve-autonomously`
- **Rationale:** Developer input in the initial run stated "everything which defines how q-block looks like should go to the convention" and "everything which defines how q-block acts/behaves/executes (all actions) should remain in aib-analyze.md." Developer correction in this run ("DP-3 resolution is not correct. Keep behavior decisions in the prompt") further clarifies that format selection guidance — the decision logic for which format to use — is behavioral and stays in the prompt. Format templates (code blocks), QID numbering rules, mandatory field rules (`Why this matters`, `*(recommended)*`), and free-text content rules are format definitions and move to the convention.
- **Resolution:** See Content Boundary Analysis tables above. Format templates and format-level rules move to `q-block-convention.md`; format selection sentences stay in `aib-analyze.md`; all other behavioral content stays in `aib-analyze.md`.
