## Goal

Merge the context-generation workflow from `.aib_brain/prompts/aib-context.md` into `.aib_brain/prompts/aib-implement.md` by copying all phases verbatim under a clearly labelled `## Context Generation Sub-flow` section. Add brownfield detection at the start of the implement workflow: when `.aib_memory/context.md` is absent or empty, the sub-flow executes automatically before any implementation work begins. Delete `aib-context.md` after the merge. Update `menu.py` to detect the empty-context workspace state and display a tooltip directing the Developer to execute `aib-implement.md` to populate context. The end state is a two-prompt AIB surface: `aib-analysis.md` and `aib-implement.md`.

## Constraints

- All phases from `aib-context.md` MUST be copied verbatim into `aib-implement.md` (Option A chosen — zero regression risk).
- `.aib_brain/` assets MUST NOT be modified by tool scripts; all changes are AI-agent edits to prompt and tool files.
- No new third-party libraries or external dependencies may be introduced.
- The existing end-of-run context update (current step 7, which runs after implementation) MUST be retained; it serves a distinct purpose (post-implementation state capture) from the brownfield start-of-run generation.
- All existing tests must continue to pass after the changes.

## Success criteria

- `.aib_brain/prompts/` contains exactly two files: `aib-analysis.md` and `aib-implement.md`; `aib-context.md` is absent.
- `aib-implement.md` contains a `## Context Generation Sub-flow` section whose content is identical to the original `aib-context.md` phases (Phase 1 through Phase 6 plus all supporting sub-sections).
- Brownfield detection: `aib-implement.md` Input Resolution includes a check after the `context.md` read step — if `context.md` is absent or empty, the Context Generation Sub-flow executes before any implementation steps.
- `menu.py` detects the empty-context state (absent or empty `.aib_memory/context.md`) and displays a guidance tooltip directing the Developer to run `aib-implement.md`.
- `python -m pytest tests/ -v` reports zero failures after all changes.
- At least one new test in `tests/test_menu.py` covers the empty-context guidance state.
