## Current Structure Issues identified so far (search others as well)

| Problem | Details |
|---------|---------|
| **Numbering drift** | Markdown headings start at "## 1. Objective" but internal references say "section 2.1", "section 4.7" etc. The numbering mixes conceptual ordering with implementation ordering. |
| **Monolithic Preflight** | Section 4 is ~200 lines containing the main flow AND two complete sub-flows (4.7, 4.8). Hard to navigate. |
| **Output specs far from generation step** | The execution flow (section 4) sends you to section 6/7 for output rules — requires jumping back and forth. |
| **Constraints scattered** | Some constraints are in "Global Constraints", some in "Analysis Requirements" (section 5), some in output contracts (section 7 invariants). |
| **Mixed abstraction** | Section 7 mixes output format definition (Plan schema, Decisions format) with procedural logic (Q-block generation rules, decision classification). |
| **Sub-flow burial** | Auto-Request Branch and Answer Application Sub-flow are nested under "Preflight" at depth 3, making them hard to reference as standalone procedures. |
