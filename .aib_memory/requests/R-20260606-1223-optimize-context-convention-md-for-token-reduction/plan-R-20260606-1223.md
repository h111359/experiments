## Goal

Update `.aib_brain/conventions/context-convention.md` to mandate a token-efficient format for `context.md` by applying 17 targeted compression changes covering statement ID format, language density, section structure, file inventory format, markdown formatting, pruning rules, and notation conventions. Update all dependent tool scripts, tests, and regenerate `context.md` to comply with the new convention.

## Constraints

- MUST follow `plan-convention.md` format and structure.
- MUST NOT modify `.aib_brain/` files other than `context-convention.md`, `tools/verify-context.py`, `tools/edit-context.py`, `tools/hash-text.py`, and `README.md` (per workspace instructions).
- All 17 input items must be reflected in the updated `context-convention.md`.
- Statement ID format after items 2+5: `- <TYPE>: text` (no hash, no area prefix); tools must use text-based line matching for statement addressing (per DP-01).
- Area headings promoted to H2 level (`## AREA`); no wrapper `## Section 2` / `## Section 3` headings (per DP-02).
- Preamble auto-generated block removed entirely (per DP-03).
- No inline markdown formatting (backticks, bold, italic) anywhere in generated `context.md` (per DP-04).
- Item 13 ("O" statement type) is a no-op — the type does not exist in the current convention.
- Tool updates (edit-context.py, verify-context.py) and test updates are in scope (per DP-06).
- `logs/next_version_changes.md` must be updated during implementation per workspace instructions.

## Success criteria

- `context-convention.md` reflects all 17 input items.
- `python -m pytest tests/test_context_formatting_rules.py` passes.
- `python -m pytest tests/test_verify_context.py` passes.
- `python .aib_brain/tools/verify-context.py --workspace .` exits with code 0 against the regenerated `context.md`.
- `python -m pytest tests/` passes (full suite).

### Task 1: Rewrite context-convention.md

#### Intent
Replace the existing `context-convention.md` with a new version that encodes all 17 compression changes while preserving the convention's normative completeness.

#### Outputs
- `.aib_brain/conventions/context-convention.md` — fully rewritten.

#### Procedure

Read the current `.aib_brain/conventions/context-convention.md` in full before writing.

Apply the following changes (mapped to input items):

**Structure and Headings (Items 1, 6):**
Remove `## Section 2 — Statements` and `## Section 3 — Workspace File Inventory` wrapper heading requirements. Update the mandatory section list to: `## 1. Product Identity`, then one `## <AREA>` heading per area (H2 level, area code only — e.g., `## FN`, `## PO`), then `## Files`. Empty area sections MUST be entirely omitted (no stub notices, no empty headings). Add explicit rule: "If a subsection has no statements, omit it entirely. Do not output empty headings or stub notices explaining missing data."

**Statement ID format (Items 2, 5):**
Replace the `<AREA>-<TYPE>-<HASH>: <text>` format with `<TYPE>: <text>`. Remove the `<HASH>` definition, computation instruction, and Uniqueness Invariant by hash. Replace with: "Statements are addressed by their type letter and text content. Tool scripts use text-based line matching for CRUD operations. Statement text MUST be unique within its area section to allow unambiguous addressing."

**Statement types (Item 13 — no-op):**
The "O" type does not exist. Retain the existing 9 types: N, R, C, E, L, U, A, D, I. Add a note in the type table: "Open/unresolved questions MUST NOT be stored in context.md. Use input.md for open questions."

**Language (Items 3, 15):**
Replace "MUST be a single grammatically complete sentence ending with a period" with: "MUST use concise, telegraphic phrasing. Omit unnecessary articles (a, an, the) and helper verbs (is, are, must be). Always use standard abbreviations: req, res, auth, db, env, config, err, msg, btn."

**Granularity vs. Grouping (Items 8):**
Update the Granularity Principle: "Each statement SHOULD express exactly one fact. Exception: tightly coupled properties of a single named entity MAY be grouped into one statement using an inline list (e.g., `- R: User table columns: email (unique), password_hash`)."

**Markdown formatting (Item 10):**
Delete formatting rule 8 ("File paths and code identifiers MUST be wrapped in backticks"). Add new rule: "Statements MUST NOT use bold, italic, or inline code formatting (backticks). Plain text only."

**Preamble (Item 9):**
Remove the mandatory preamble block (`> **Auto-generated** by...`). State: "No preamble or version metadata is required. Version tracking is via Git."

**Pruning (Item 7):**
Add a new Pruning Rule section: "Aggressively delete historical context, past design iterations, and resolved open questions. Do not retain deprecated features. context.md must represent only the CURRENT state of the application."

**Aliases (Item 11):**
Add an optional Aliases section before `## 1. Product Identity`: "If the workspace has long repeated names, define short aliases at the top of the file. Format: `Aliases: SHORT=Long Name, ...`. Use the short alias exclusively in all statements."

**No negative space (Item 16):**
Add rule: "Do not state standard defaults (e.g., API returns JSON, uses REST). Do not document negative space (e.g., No caching implemented) unless it overrides a standard expectation. Document only custom business logic and deviations."

**Logic notation (Item 14):**
Add rule: "For logic and flow statements, prefer pseudo-code and logic operators (`->`, `&&`, `||`, `==`, `!=`) over natural language when the expression is shorter."

**File inventory (Items 4, 12):**
Replace the flat bullet format for Section 3 (now `## Files`) with an indented tree format. Change the heading from `## 3. Workspace File Inventory` to `## Files`. Add rule: "Only list directories and core architectural files. Ignore tests, config files, assets, and minor utilities unless they contain critical business logic." Update the format example to show indented tree structure.

**Subsection list update (Item 6):**
Replace the 22 `### 2.N Area — AREA` headings with 22 `## AREA` headings (two-letter area code only). List them in the same order. Remove the numbering and long labels.

**Formatting rules update:**
Update the heading hierarchy rule to: H1 for document title only (`# Product Context`). H2 for area sections and `## Files`. H3 MAY be used for sub-areas. No H4 or deeper.

**Remove table prohibition adjustment:**
Keep the no-tables rule in effect (routing/endpoints use indented block format, not Markdown tables, per item 17 examples). Update the routing/endpoints section to add: "API routes and endpoints MUST be represented as an indented block under a type statement, not as Markdown prose or tables."

#### Done criteria

- `context-convention.md` no longer contains `<AREA>-<TYPE>-<HASH>`.
- `context-convention.md` no longer contains `## 2. Statements` or `## 3. Workspace File Inventory`.
- `context-convention.md` contains the telegraphic phrasing rule.
- `context-convention.md` contains the pruning rule.
- `context-convention.md` contains the no-formatting rule (no backticks in statements).
- `context-convention.md` contains the no-negative-space rule.
- `context-convention.md` contains the Aliases section definition.
- Statement format defined as `<TYPE>: <text>`.
- All 9 statement types (N, R, C, E, L, U, A, D, I) still defined.

#### Dependencies
None.

#### Risk notes
- This is a full rewrite of the convention. Care must be taken not to omit any existing normative rule that is not explicitly overridden by the 17 items (e.g., no-HTML rule, no-external-hyperlinks rule).
- The no-tables rule must be retained since item 17's examples use indented blocks, not Markdown tables.

### Task 3: Update edit-context.py

#### Intent

#### Outputs
- `.aib_brain/tools/edit-context.py` — updated addressing logic.

#### Procedure

Read the current `.aib_brain/tools/edit-context.py` in full.

Remove all calls to `hashlib` for statement ID generation.

Update uniqueness validation: before inserting, check that no existing statement in the area section has identical text content (case-insensitive). Reject duplicates.

Update the `--select` operation to return the full matching statement line.

Update all error messages to reference the new format.

#### Dependencies
