## Goal

Rename the `## Active request` section heading to `## Status` in all `input.md` seed templates across AIB tooling, add a `State: <value>` tracking line (per-tool defaults: `analysis_ready` for finalize-input.py, `idle` for close-request.py and initialize.py), update `aib-analyze.md` to override state to `questions_generated` when Q-blocks are generated and to emit a short step-completion note after each of steps S01â€“S10, and update all tests that reference the old heading or seed template constants.

## Constraints

- State values are exactly the four specified: `idle`, `analysis_ready`, `implementation_ready`, `questions_generated`. No new values may be introduced.
- `menu.py` display label `"Active request: {req_text}\n"` must NOT be modified â€” it is a UI string, not the `input.md` section heading.
- Archived input files under `.aib_memory/requests/*/inputs/` and `.aib_memory/archives/` must NOT be modified (GC-01, GC-04).
- `aib-implement.md` must NOT be modified; `implementation_ready` is reserved for future use.
- The `_is_stub_equivalent` function in `finalize-input.py` must be updated so that the new two-variable-line structure under `## Status` (request ID line + State line) does not cause false stub-equivalence failures when comparing `input.md` instances with different state values.
- All changes to `.aib_brain/` files are authorized as direct implementation targets for this request.

## Success criteria

- `input.md` reset by `finalize-input.py` contains `## Status` heading, the active request ID line, and `State: analysis_ready`.
- `input.md` reset by `close-request.py` contains `## Status`, `No active request`, and `State: idle`.
- `input.md` seeded by `initialize.py` contains `## Status`, `No active request`, and `State: idle`.
- `_is_stub_equivalent` returns `True` when comparing two `input.md` instances that differ only in the request ID line or the `State:` line value.
- `aib-analyze.md` contains a step-completion note sub-step at the end of each step S01â€“S10 in the format `[S0N done] <one-sentence outcome summary>.`
- `aib-analyze.md` S08 contains an instruction to directly edit `input.md` to replace `State: analysis_ready` with `State: questions_generated` when Q-blocks are written.
- All tests in `test_finalize_input.py`, `test_close_request.py`, `test_initialize.py`, and `test_menu.py` pass after updates.
- `context.md` is updated to reflect the renamed section, the new `State:` tracking line, and the per-step notes.
