## Goal

Fix the GitHub Actions CI workflow so it no longer fails when a PR branch carries a manually-applied MINOR or MAJOR SemVer marker. Update `scripts/release_bookkeeping.py` to accept any head version greater than the base version as a valid pre-bumped target, and update the workflow display name to reflect this broadened scope.

## Constraints

- Must not break the existing PATCH auto-bump behavior (when head marker equals base marker, CI must still auto-apply a PATCH increment).
- Must not rename the workflow file `aib-semver-patch-bump-and-log.yml` (referenced by tests).
- All existing tests must continue to pass.
- Changes are limited to `scripts/release_bookkeeping.py`, `.github/workflows/aib-semver-patch-bump-and-log.yml`, `tests/test_release_bookkeeping.py`, `logs/next_version_changes.md`, and `.aib_memory/context.md`.

## Success criteria

- Running `python -m pytest tests/test_release_bookkeeping.py tests/test_semver_workflow_structure.py` exits with code 0 and reports 19 passed.
- `scripts/release_bookkeeping.py` `Version` dataclass carries `order=True`.
- The mismatch check in `release_bookkeeping.py` raises an error only when `head_version < base_version`; it does not raise for `head_version > base_version.bump_patch()`.
- `.github/workflows/aib-semver-patch-bump-and-log.yml` `name:` reads `AIB SemVer bump and log`.
- `logs/next_version_changes.md` contains curated change bullets for this request.

## Plan

### Task 1: Fix version comparison in release_bookkeeping.py

#### Intent
Replace the PATCH-only mismatch guard with version-comparison branching that accepts any pre-bumped head version greater than the base.

#### Outputs
`scripts/release_bookkeeping.py` — updated `Version` dataclass decorator and revised mismatch check block.

#### Procedure
1. In `scripts/release_bookkeeping.py`, add `order=True` to `@dataclass(frozen=True)` on the `Version` class, producing `@dataclass(frozen=True, order=True)`.

2. In `scripts/release_bookkeeping.py`, replace the block beginning with `target_version = base_version.bump_patch()` through the `if head_marker not in (base_marker, target_marker): raise ReleaseBookkeepingError(...)` with version-comparison branching: parse `head_version = Version.parse_marker(head_marker)`; raise error if `head_version < base_version`; set `target_version = base_version.bump_patch()` if equal; set `target_version = head_version` if greater.

#### Done criteria
- `order=True` present on `Version` dataclass in `scripts/release_bookkeeping.py`.
- Old `if head_marker not in (base_marker, target_marker)` guard removed.
- New branching logic present: `head_version < base_version` → error, `== base_version` → patch bump, `> base_version` → use head.

#### Dependencies
None.

#### Risk notes
`order=True` uses Python tuple comparison on `(major, minor, patch)` in declaration order — semantically correct for SemVer numeric precedence because fields are declared in that order.
