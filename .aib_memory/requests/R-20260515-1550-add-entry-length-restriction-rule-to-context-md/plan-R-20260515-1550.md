## Goal

Add an explicit two-sentence entry-length enforcement bullet to the `aib-context.md` Phase 4 formatting checklist, and add test coverage for that rule in `tests/test_context_formatting_rules.py`, ensuring all future auto-generated `context.md` files reliably produce concise, human-readable bullet entries.

## Constraints

- `.aib_brain/conventions/context-convention.md` already contains rule 16 (two-sentence limit). No convention change is required.
- `.aib_memory/context.md` is auto-generated by `aib-context.md`; direct edits will be overwritten on the next prompt run. Remediating existing long entries requires a separate `aib-context.md` execution, which is not part of this implementation run.
- All changes to `.aib_brain/` files are permitted for AIB framework assets within this request's scope.
- The new checklist bullet must match the style of existing Phase 4 checklist bullets in `aib-context.md`.

## Success criteria

1. `.aib_brain/prompts/aib-context.md` Phase 4 "Formatting requirements" block contains an explicit bullet stating that each bullet or list item MUST NOT exceed two sentences.
2. `tests/test_context_formatting_rules.py` class `TestContextConventionFormattingRules` contains a test method that asserts the two-sentence limit rule is present in `context-convention.md`.
3. `tests/test_context_formatting_rules.py` class `TestContextPromptFormattingChecklist` contains a test method that asserts the two-sentence enforcement text is present in `aib-context.md`.
4. `pytest tests/test_context_formatting_rules.py -v` reports all tests passing.
5. `logs/next_version_changes.md` contains a bullet entry describing this change.
