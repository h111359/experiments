## Goal

Remove the `.aib_brain/templates/` folder and its single file (`requests_register-template.md`) from the AIB framework. Replace the template-file-based seeding of `requests_register.md` with a hardcoded string constant inside `initialize.py`, consistent with how all other `.aib_memory/` artifacts are already seeded. Remove the now-dead `load_template` helper from `common.py` and all associated test coverage.

## Background

The `.aib_brain/templates/` directory currently holds one file: `requests_register-template.md`. This file is read by `initialize.py` via `load_template()` to seed `.aib_memory/requests_register.md` on workspace initialization. Every other artifact seeded by `initialize.py` (`context.md`, `input.md`, `instructions.md`) is already produced from hardcoded inline strings — the template file is the sole exception.

The register schema and validation rules are fully defined in the existing `.aib_brain/conventions/requests_register-convention.md`, making the template file redundant as a definition source. Eliminating the external file dependency simplifies the brain asset set, removes one failure mode (`load_template` raising `ValidationError` when the file is absent), and aligns `initialize.py` with the hardcoding pattern already used for all other seeds.

## Scope

- Replace `load_template(brain_dir, "requests_register-template.md")` call in `.aib_brain/tools/initialize.py` with a hardcoded inline string that produces the same seed content (`# Requests Register` title + empty Markdown table with the required columns).

- Remove the `load_template` import from `.aib_brain/tools/initialize.py`.

- Remove the `load_template` function from `.aib_brain/tools/common.py`.

- Delete `.aib_brain/templates/requests_register-template.md`.

- Delete the `.aib_brain/templates/` folder (which becomes empty after the file is removed).

- Update `tests/conftest.py`: remove the `TEMPLATES_DIR` constant and the template-copying logic inside `_seed_workspace`.

- Update `tests/test_initialize.py`: remove the `TEMPLATES_DIR` constant and all template-copying code inside helper functions.

- Update `tests/test_lifecycle_e2e.py`: remove the `TEMPLATES_DIR` constant and all template-copying code inside `_make_brain_only_workspace`.

- Remove the `TestLoadTemplate` test class and its `load_template` import from `tests/test_tools_common.py`.

- Update the informational upgrade-prompt text in `.aib_brain/tools/menu.py` that references "brain templates" to instead say "brain assets".

- Update `.aib_memory/context.md` to remove or revise the "AIB Templates" component entry.

## Out of scope

- Changing the content of any existing `.aib_memory/requests_register.md` file.

- Modifications to `.aib_brain/conventions/requests_register-convention.md`.

- Changes to how `create-request.py`, `close-request.py`, or any other tool script reads or writes the register.

- Adding a new convention file for the register (one already exists).

## Constraints

- Tool scripts must use the Python standard library only (NFR-004); no new dependencies introduced.

- The hardcoded seed string must produce functionally identical content to the former template file (same column names, same heading).

- The idempotent behaviour of `initialize.py` (skip when file already exists) must be preserved unchanged.

- All existing tests that exercise `initialize.py` seeding must continue to pass after removing the template-copy step from test fixtures.

## Success criteria

- `initialize.py --workspace <path>` succeeds and creates a valid `requests_register.md` without any `.aib_brain/templates/` folder present.

- No file in the workspace contains a reference to `.aib_brain/templates/` or `requests_register-template.md` (verified by grep).

- The `.aib_brain/templates/` folder does not exist in the repository.

- `load_template` is absent from `common.py` and all tool scripts.

- The full test suite passes: `pytest tests/` exits 0 with no failures or errors.

## Assumptions

- A1: The seed content that `initialize.py` should hardcode is exactly the title line plus the empty Markdown table (matching the template's table-only portion, not including the "Validation rules" commentary block, which belongs in the convention file).
  - Risk if false: The seeded register file may differ from historical format, causing parse failures in scripts that rely on specific column order or heading presence. Mitigation: the `parse_markdown_table` helper in `common.py` is robust to extra non-table content.

- A2: No other file in `.aib_brain/` or `tests/` depends on the `load_template` function beyond the two currently identified callers (`initialize.py` and `test_tools_common.py`).
  - Risk if false: Removing `load_template` breaks an undiscovered call site. Mitigation: grep confirms two callsites only.

- A3: The informational text in `menu.py` ("brain templates") is user-visible and should be updated for accuracy, but the change carries no functional risk.
  - Risk if false: No functional risk; purely cosmetic accuracy.
