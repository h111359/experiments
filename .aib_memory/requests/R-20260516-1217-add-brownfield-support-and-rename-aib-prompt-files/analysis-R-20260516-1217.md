## Input Interpretation

The developer requests three related, concurrently deliverable changes to the AIB framework:

**Change 1 — Brownfield auto-context detection.** The renamed analysis prompt (`aib-analyze.md`) shall detect, before any preflight step, whether `.aib_memory/context.md` is absent or contains only whitespace. When that condition holds, the prompt instructs the AI agent to execute `.aib_brain/prompts/aib-refresh-context.md` first, then resume the normal analysis flow. This unblocks developers applying AIB to an existing repository without having previously run context generation. The detection step must not introduce a recursive loop — the context prompt must not itself invoke the analysis prompt.

**Change 2 — Menu advisory update.** `menu.py`'s `_is_context_empty`-gated advisory line currently instructs the developer to run context generation manually. The developer requests the wording be updated to state that context gathering will execute automatically when the analysis prompt is invoked, removing the manual prerequisite step.

**Change 3 — Prompt renames.** `aib-analysis.md` is renamed to `aib-analyze.md` and `aib-context.md` is renamed to `aib-refresh-context.md`. Q&A resolved: `aib-analyze.md` (Q001, Option A) and `aib-refresh-context.md` (Q002, Other — developer-entered). All workspace references to the old filenames — in test path constants, README instructions, `aib-implement.md`, and `menu.py` guidance strings — are updated atomically with the rename.

### Industry Knowledge

**Action-verb naming for tools (Unix/POSIX tradition; GNU tooling guidelines):** Command-line tools and their sub-commands follow verb-first naming (`git commit`, `make build`, `docker push`). Applied to AIB prompt files, action-verb names make the developer's intent explicit at the call site and improve discoverability. Applicability: high — directly motivates the rename from noun-based (`analysis`, `context`) to verb-based (`analyze`, `refresh-context`) filenames.

**Progressive disclosure / "Pit of Success" pattern (.NET design guidelines; Rico Mariani):** Tools should make the correct path the path of least resistance. Auto-detecting a missing prerequisite (empty `context.md`) and resolving it transparently, rather than failing or asking for manual intervention, reduces friction for new users and eliminates an entire class of onboarding errors. Applicability: high — directly justifies the brownfield auto-detection feature.

**Atomic rename in VCS-tracked projects (Git documentation, open-source contribution guides):** Renaming files in a version-controlled workspace requires updating all consumers in the same commit to avoid broken references between commits. Standard procedure: rename, update all referencing files atomically, verify tests before commit. Applicability: high — governs the implementation sequencing (single-pass atomically per Task 1–8 ordering).

### Feasibility Notes

All three changes are mechanically straightforward. The brownfield detection step is an additive instruction insert with no tool script dependency. The menu message update is a single multi-line string replacement. The renames are wide-ranging across approximately seven files plus auto-regenerated `context.md`, but each update is a deterministic string substitution. The highest-risk item is the `test_menu.py` assertion: if the new advisory text does not contain the string the test expects, the test will fail. The chosen advisory wording from Task 3 must be coordinated with the Task 6 assertion update.

**Non-recursion:** Confirmed by inspection — `aib-context.md` (to become `aib-refresh-context.md`) does not invoke `aib-analysis.md` or `aib-analyze.md`. No loop risk.

## Request Context

### Background

AIB is optimized for greenfield usage — workspaces where `context.md` was populated during initialization. Developers applying AIB to an existing repository (brownfield scenario) encounter a friction point: the analysis prompt runs without product context unless the developer remembers to run `aib-context.md` first. Additionally, the current prompt filenames `aib-analysis.md` and `aib-context.md` follow a noun-based convention inconsistent with `aib-implement.md` (action-verb). This inconsistency creates cognitive friction when developers learn the toolset.
