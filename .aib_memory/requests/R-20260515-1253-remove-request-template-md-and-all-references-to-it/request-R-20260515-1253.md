## Goal

Remove `.aib_brain/templates/request-template.md` and all non-archive references to it in the active workspace. After this change, the authoritative source of `request.md` structure is exclusively `request-convention.md` and `aib-analysis.md`.

## Background

Prior to v1.2.0, `create-request.py` seeded a stub `request.md` in the request folder using `request-template.md`. ADR-0006 removed that behaviour: `create-request.py` now creates only the folder and register row, and `request.md` is AI-generated in full by `aib-analysis.md` from `input.md` content, structured according to `request-convention.md`. The template is therefore dead code — not read by any tool script, prompt, or test — yet still present in `.aib_brain/templates/` and referenced in two bullets in `.aib_memory/context.md`. Its continued presence creates a false impression that it plays a role in request creation.

## Scope

- Delete `.aib_brain/templates/request-template.md`.

- Remove the `request-template.md` bullet from the **Module Breakdown / Templates** section of `.aib_memory/context.md` (the bullet starting with `` `request-template.md` — Legacy seed template...``).

- Remove the `.aib_brain/templates/request-template.md` bullet from the **Workspace File Inventory** section of `.aib_memory/context.md` (the bullet starting with `` `.aib_brain/templates/request-template.md` — Legacy seed template for `request.md`...``).

## Out of scope

- Archive files under `.aib_memory/archives/` — immutable historical snapshots; not modified.

- `requests_register-template.md` and any other file in `.aib_brain/templates/` — only `request-template.md` is being removed.

- Any refactoring of `initialize.py`, `create-request.py`, or any other tool script.

- Re-running `aib-context.md` to regenerate `context.md` from scratch — only the two targeted bullet entries are removed.

## Constraints

- `request-convention.md` MUST remain unchanged and intact as the normative structural authority.

- The edit to `.aib_memory/context.md` MUST be surgical: only the two identified bullet lines are removed; no adjacent content is modified.

- `.aib_brain/` is normally not modified by tool scripts (ADR-0003); in this case the deletion is performed directly by the AI agent as an explicit framework asset removal authorised by this request.

- The test suite (`pytest tests/`) MUST pass after the change with no new failures.

## Success criteria

- `.aib_brain/templates/request-template.md` does not exist in the workspace.

- A workspace-wide search for `request-template` returns no results in non-archive files (i.e., no matches outside `.aib_memory/archives/`, `.aib_memory/requests/*/inputs/`, or `.aib_memory/requests_register.md` title column).

- `.aib_memory/context.md` no longer contains either of the two identified bullet entries.

- `pytest tests/` exits with code 0 and no new test failures compared to pre-change baseline.

## Assumptions

- A1: Priority is normal (non-blocking cleanup); no urgency constraint exists.
  - Risk if false: if this is blocking an upgrade or release, the plan should be accelerated.

- A2: Archive files under `.aib_memory/archives/` and request input archives are intentionally excluded from "all references" — they are immutable historical records.
  - Risk if false: if archives must also be cleaned, scope expands significantly.

- A3: No tool script, prompt, or test references `request-template.md` at runtime (confirmed by grep scan).
  - Risk if false: a missed reference would cause a runtime error after deletion.

- A4: `context.md` is the only non-archive active-workspace file with live references (confirmed by grep scan).
  - Risk if false: additional references would remain after the edit.

- A5: The success criteria for "all references removed" excludes the `requests_register.md` title column entry for this request itself (the request title contains the phrase "request-template.md").
  - Risk if false: ambiguity about what counts as a "reference."
