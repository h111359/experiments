## Goal

**Background:** The current AIB implement prompt reads `.aib_memory/context.md` at startup (Step 4) and uses its content as a global context source throughout the implementation run. This creates an implicit coupling between the plan artifact and the workspace memory store: a plan written today may produce a different implementation tomorrow if `context.md` is updated independently. It also means the plan cannot be handed to a human engineer or a fresh AI session without providing `context.md` alongside it.

**What must be achieved:** The plan artifact (`plan-<request_id>.md`) must become the single authoritative source for everything needed during implementation. The `## Goal` section of each plan must include the full background (why the change is needed and which components are affected). Every procedure step in every plan task must reference the exact file path it operates on. Context update tasks must embed the exact `edit-context.py` invocations with literal command arguments. The `aib-implement.md` prompt must remove the `context.md` read step and explicitly forbid reading `context.md` during implementation. `plan-convention.md` must codify these requirements normatively and remove the `#### Risk notes` sub-field from the task schema.

**Impacted components:** `.aib_brain/prompts/aib-analyze.md`, `.aib_brain/prompts/aib-implement.md`, `.aib_brain/conventions/plan-convention.md`, `.aib_brain/README.md`, `.aib_memory/context.md`, `tests/test_analysis_prompt_structure.py`, `tests/test_tools_common.py`.

## Constraints

- MUST NOT modify any files outside the listed scope.
- MUST NOT break currently passing tests. Pre-existing failures (identified in pre-flight) must not be worsened.
- Removal of `#### Risk notes` from `plan-convention.md` must not break the `validate_plan_md` check in `tests/test_tools_common.py` (Risk notes is not a validated mandatory section — only the four top-level sections are validated).
- `aib-implement.md` Rules section must still permit reading convention files (e.g., `context-convention.md`) before editing `context.md`. Only reading `context.md` itself is forbidden.
- Step numbering in `aib-implement.md` must be consistent after removing Step 4 — all subsequent steps renumbered.

## Success criteria

- `plan-convention.md` contains no `#### Risk notes` sub-field in the task schema template.
- `plan-convention.md` explicitly states that implement MUST NOT read `context.md`.
- `plan-convention.md` mandates that context update tasks include exact `edit-context.py` invocations.
- `aib-implement.md` contains no step that reads `context.md`.
- `aib-implement.md` Rules section contains a prohibition on reading `context.md` during implementation.
- `aib-analyze.md` Step 9 mandates plan self-sufficiency and exact `edit-context.py` invocations in context update tasks.
- `tests/test_tools_common.py` VALID_PLAN_MD fixture does not contain `#### Risk notes`.
- `tests/test_analysis_prompt_structure.py` contains at least two new tests asserting the context.md prohibition in `aib-implement.md` and the Risk notes removal from `plan-convention.md`.
- All tests that were passing before this change continue to pass after.
