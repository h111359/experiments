## Goal

Restructure the `analysis-<request_id>.md` document format as defined in `analysis-convention.md` and generated by `aib-analyze.md`. The changes are: (1) replace `## Executive Summary` and `## Request Context` with a new `## Overview` section placed first in the section order, containing Request ID, Request title, Background, Scope, and Out of scope; (2) replace `### Feasibility Notes` and `### Expert Observations` with `### AI Agent critique`, a broader holistic workspace review sub-section within `## Research Results`; (3) restructure `## Implementation Alternatives` to use per-decision-point format with named alternative approaches per fork; (4) convert the Decision Points markdown table to a heading/sub-heading list structure. All changes are applied to `analysis-convention.md`, `aib-analyze.md`, and supporting test and documentation files.

## Constraints

- GC-07 applies during the analysis run: only `.aib_memory/` writes are permitted during analysis. All `.aib_brain/` changes are deferred to the implementation run.

- Existing tests in `tests/test_analysis_prompt_structure.py` must continue to pass after changes. Specifically: `test_implementation_alternatives_present`, `test_decision_points_catalog_required_in_prompt`, `test_execution_model_summary_present`, `test_global_constraints_present`, `test_failure_handling_present`, `test_gc06_no_closed_request_reads_present`, and all other pre-existing tests must remain green.

- The stale `aib-analysis.md` reference in `plan-convention.md` is out of scope for this request and must not be fixed here.

- The `analysis-convention.md` section numbering gap (sections 6–10 absent) is pre-existing and out of scope.

- `context.md` is auto-generated; manual edits are not needed — updating convention and prompt files plus running `aib-refresh-context.md` is the correct update path.

- All `.aib_brain/` file edits must be made during `aib-implement.md` execution; this plan MUST NOT be used to justify edits outside that context.
