## Goal

Extend Q-block generation in `aib-analysis.md` so that each Q-block written to `.aib_memory/input.md ## Questions` marks the recommended option with `*(recommended)*`. The AI must place the recommended option first in the list. The Answer Application Sub-flow tie-break rule must be updated to apply the recommended-marked option (rather than the first listed option) when a question is left unanswered.

## Constraints

1. The change is limited to `.aib_brain/prompts/aib-analysis.md` and `.aib_memory/context.md`; no tool script changes are needed.
2. Existing prompt structural conventions (section numbering, heading levels) must be preserved.
3. The marker string is `*(recommended)*` — no alternative format is introduced.
4. Exactly one option per Q-block is marked; all other options carry no marker.
5. The change applies only to new Q-block generation; archived Q-blocks in closed request folders are not retroactively changed.

## Success criteria

1. After `aib-analysis.md` generates a Q-block, exactly one option in the block is marked `*(recommended)*`.
2. The `*(recommended)*` marker appears on the first listed option (the AI's preferred choice) in every generated Q-block.
3. The Answer Application Sub-flow applies the recommended-marked option as the default when a question is left unanswered.
4. All existing automated tests pass; `test_analysis_references_recommended_marker` in `tests/test_questions_in_input_md.py` is updated to assert the new rule.
5. `context.md` FR-007 reflects the updated Q-block format and tie-break behavior.

## Decisions

**Q001:** How should the recommended option be identified in a Q-block — by AI judgment (AI picks the best option and marks it regardless of list position) or by structural convention (the first listed option is always the recommended one)? → **Chosen:** Place first the recommended but still mark it with `*(recommended)*`

**Q002:** Should the Answer Application Sub-flow tie-break behavior change from "apply first listed" to "apply recommended-marked option" when a question is left unanswered? → **Chosen:** Yes — change the tie-break to apply the recommended-marked option. Marker and default behavior are always aligned.
