## Goal

Restructure the AIB knowledge representation model by replacing the prose-based `context.md` format with an atomic statement format, creating supporting CRUD tooling (`hash-text.py` and `edit-context.py`), updating the context convention, and modifying the analyze and refresh-context prompts to integrate with the new format.

## Constraints

- Python 3 must be used for all tool scripts; no additional dependencies beyond the standard library.
- The `.aib_brain/` folder modification is authorized for this request (new scripts in `tools/`, convention file update, prompt updates).
- The hash must produce exactly 8 hexadecimal characters using truncated SHA-256.
- Statement index (area+type+hash) must be unique within `context.md`.
- Sections `## 1. Product Identity` and `## 3. Workspace File Inventory` retain their current prose format.
- The `edit-context.py` script must validate uniqueness on every insert operation.
- Existing tests in `tests/test_context_formatting_rules.py` will require updating to match the new format.

## Success criteria

- `hash-text.py` accepts a text string argument and outputs an 8-character hex hash (truncated SHA-256) to stdout.
- `edit-context.py` supports `select`, `insert`, and `delete` operations with appropriate arguments (area, type, hash, text) and validates uniqueness.
- `context-convention.md` defines the new 3-section structure (Product Identity, Statements with 22 subsections, Workspace File Inventory) and the atomic statement format.
- `aib-analyze.md` Step 9 includes context-update tasks in the generated plan WBS.
- `aib-refresh-context.md` includes enrichment passes that verify analysis decisions, plan results, and modified file changes are reflected in context.
- `aib-refresh-context.md` detects old-format `context.md` and converts using existing content as input.
- All new scripts exit 0 on success and non-zero on error with descriptive stderr messages.

## Plan

### Task 1: Create hash-text.py script

#### Intent

Create the `hash-text.py` tool script that generates an 8-character hex hash from input text.

#### Outputs

`.aib_brain/tools/hash-text.py`

#### Procedure

Create `.aib_brain/tools/hash-text.py` with the following behavior:

- Accept a single positional argument (the text to hash).

- Compute `hashlib.sha256(text.encode('utf-8')).hexdigest()[:8]`.

- Print the 8-character hash to stdout and exit 0.

- On missing argument, print usage to stderr and exit 1.

#### Done criteria

- Running `python .aib_brain/tools/hash-text.py "The system must download only files that are new."` outputs an 8-char hex string to stdout.
- Running without arguments exits with code 1 and prints usage to stderr.

#### Dependencies

None

#### Risk notes

None

### Task 2: Create edit-context.py script

#### Intent

Create the `edit-context.py` tool script that performs select, insert, and delete operations on atomic statements in `.aib_memory/context.md`.

#### Outputs

`.aib_brain/tools/edit-context.py`

#### Procedure

Create `.aib_brain/tools/edit-context.py` with the following operations:

- `--operation select --area <prefix> --type <letter> --hash <hash>` — Find and print the statement text matching the given area+type+hash index. Exit 0 if found, exit 1 if not found.

- `--operation insert --area <prefix> --type <letter> --hash <hash> --text "<statement text>"` — Insert a new atomic statement in the appropriate subsection of `## 2. Statements`. Validate that no duplicate index exists (exit 1 with error if duplicate). Optionally accept `--related "<comma-separated identifiers>"` to append the relationship suffix.

- `--operation delete --area <prefix> --type <letter> --hash <hash>` — Remove the atomic statement matching the index. Exit 0 if removed, exit 1 if not found.

- Accept `--workspace <path>` argument (default `.`) to locate `context.md` at `<workspace>/.aib_memory/context.md`.

- Validate that the target section `## 2. Statements` and appropriate subsection exist before insert. If the subsection does not exist, create it in the correct order.

- After any modification, re-validate uniqueness across the entire file.

#### Done criteria

- Insert operation creates a correctly formatted atomic statement line in the appropriate subsection.
- Select operation returns the matching statement text.
- Delete operation removes the matching line.
- Inserting a duplicate index exits with code 1 and prints an error message.
- All operations handle missing `context.md` gracefully with appropriate error messages.

#### Dependencies

Task 1 (hash-text.py provides the hash generation logic that edit-context.py can reference)

#### Risk notes

The script must handle concurrent-write scenarios gracefully (file read-modify-write race). Since AIB is single-agent, this is low risk.
