## Goal

Enforce strict ordering in `aib-analysis.md` so that `request-<request_id>.md` is generated only after all clarifying questions have been answered. When the Auto-Request Creation Branch generates Q-blocks, the request file MUST NOT be created in the first pass — it is created in full on re-run after Q&A is resolved. A new `## Input Interpretation` mandatory section in `analysis-<request_id>.md` provides the authoritative source material for constructing `request.md` on re-run.

## Background

The Auto-Request Creation Branch of `aib-analysis.md` currently writes `request-<request_id>.md` (all 10 sections, with sections 1–6 derived from raw `input.md` content) and generates Q-blocks in the same execution pass. This causes the AI to anchor its interpretation of developer intent in `request.md` before Q&A is resolved. The Answer Application Sub-flow on re-run then operates on this anchored document, making it less likely to surface or correct misinterpretations. The developer's requirement is that `request.md` must reflect confirmed, Q&A-resolved intent — not first-pass AI assumptions.

## Scope

- The ordering rule in `aib-analysis.md` section 4.7 (Auto-Request Creation Branch) that governs when `request-<request_id>.md` is written: step 5 (file generation) is removed from the branch; the file is NOT written in the first pass when Q-blocks are generated.

- Preflight step 3 adaptation (`aib-analysis.md` section 4.3): the halt condition for absent `request.md` (GC-04) must be relaxed when the deferred-creation state is detected (i.e., `input.md` has a `## Questions` section and `request.md` is absent).

- The Answer Application Sub-flow (`aib-analysis.md` section 4.8): extended with a step that creates `request.md` from scratch when it is absent, using the `## Input Interpretation` section from the existing `analysis-<request_id>.md` plus Q&A answers and the request title.

- A new `## Input Interpretation` mandatory section in `analysis-<request_id>.md`: generated in every analysis run (first pass and re-run); contains an AI-rewritten, terminology-correct interpretation of the developer's `## Input` content, enriched by `context.md` product knowledge and relevant external knowledge. This section becomes the source material for populating sections 1–6 of `request-<request_id>.md` on re-run.

- `analysis-convention.md`: updated to add `## Input Interpretation` as the 3rd mandatory section (between `## Files Read During This Analysis Run` and `## Research Results`).

- Section 6 of `aib-analysis.md` (Output Contract — Analysis Document): generation instructions added for `## Input Interpretation`.

- `tests/test_analysis_prompt_structure.py`: new structural tests added asserting the deferral rule language and `## Input Interpretation` reference are present in `aib-analysis.md` and `analysis-convention.md`.

- `context.md`: FR-003, FR-004, and AC-3 updated to reflect the new first-pass / re-run behavior and the new mandatory analysis section.

## Out of scope

- Changes to `aib-implement.md` or any other prompt file not directly involved in request creation and Q-block ordering.

- Changes to tooling scripts (`create-request.py`, `finalize-input.py`, etc.) unless strictly necessary to support the ordering change. Based on resolved Q&A, no tooling script changes are needed.

- UI or CLI changes.

## Constraints

- The fix MUST NOT break the existing single-cycle Q&A contract: one round of questions, one re-run to apply answers.

- Backwards compatibility with already-active requests (requests that already have a `request-<request_id>.md`) must be preserved — the new ordering applies only to the Auto-Request Creation Branch path.

- The seed `request-<request_id>.md` created by `create-request.py` (the folder scaffold entry, not the content file) must still be created immediately so the register has a valid entry. Only the content file at `.aib_memory/request-<request_id>.md` is deferred.

- The `"All 10 mandatory sections"` phrase must remain present in `aib-analysis.md` after the restructure (relocated to the Answer Application Sub-flow's `request.md` creation instructions) to preserve the existing test assertion.

## Success criteria

- On the first analysis run when Q-blocks are generated, `.aib_memory/request-<request_id>.md` does NOT exist at the end of the run.

- On the re-run after all Q-blocks are answered, `.aib_memory/request-<request_id>.md` is created for the first time with all 10 mandatory sections fully populated.

- No regression in the standard flow (existing Active requests, no Q-blocks path): `request-<request_id>.md` is written normally in a single pass.

- `analysis-<request_id>.md` contains a `## Input Interpretation` section in every run (first pass and re-run).

- All relevant tests in `test_analysis_prompt_structure.py` pass after the implementation.

## Assumptions

- A1: The change is limited to prompt text in `.aib_brain/prompts/aib-analysis.md` and convention text in `.aib_brain/conventions/analysis-convention.md`. No tooling script changes are required.
  - Risk if false: If the stub detection mechanism requires script-level state, `finalize-input.py` or `create-request.py` may need minor adjustments, widening scope.

- A2: The deferred-creation state is reliably detected by checking two conditions simultaneously: `request.md` absent at `.aib_memory/request-<request_id>.md` AND `## Questions` section present in `input.md`. No additional flag or marker file is needed.
  - Risk if false: If there are edge cases where `request.md` is absent for reasons other than deferral (e.g., manual deletion), the deferred-creation path could be triggered incorrectly. Acceptable risk given current workspace discipline.

- A3: `test_analysis_prompt_structure.py` class `TestTenSectionMandatoryListInAnalysisPrompt` asserts `"All 10 mandatory sections" in content` — this test will continue to pass as long as that exact string appears anywhere in `aib-analysis.md` after the restructure. The string will be relocated to the Answer Application Sub-flow `request.md` creation instructions.
  - Risk if false: If the string must appear in the auto-request creation branch specifically (not just anywhere in the prompt), a test regression occurs requiring a test update.

- A4: The Answer Application Sub-flow (section 4.8) can be extended with a `request.md` creation step without requiring a new sub-flow. The step runs first (before applying Q-block answers to sections) and produces the file from Input Interpretation + Q&A answers + request title.
  - Risk if false: If the creation logic is complex enough to warrant isolation, a separate sub-flow may be needed — adding complexity beyond the current estimate.

- A5: The `## Input Interpretation` section generated in the first pass contains sufficient information to reconstruct sections 1–6 of `request.md` on re-run, even after `input.md` is archived. The section is generated from in-memory input content (read before finalize-input.py resets `input.md`), so the original developer text is preserved in the analysis artifact.
  - Risk if false: If the Input Interpretation is too abstract and loses key developer intent details, sections 1–6 on re-run may be less accurate. Mitigated by writing the section verbatim from the developer's input text, enriched (not replaced) by product terminology.
