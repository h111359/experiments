## Research Results

### Workspace Findings

**Root cause — `move-request-artifacts.py` (lines 88–95):**

The function `move_artifacts` constructs the destination path using a bare name:

```python
bare_name = f"{artifact_type}.md"
dest = dest_folder / bare_name
```

This unconditionally strips the request ID suffix from every artifact type (`request`, `analysis`, `UAT_scenarios`) before moving it into the archive subfolder. The variable `filename` (which already holds the correct ID-suffixed value, e.g. `request-R-20260514-2159.md`) is computed earlier in the loop but not used for the destination. The fix is a single-line change: replace `dest = dest_folder / bare_name` with `dest = dest_folder / filename` and remove the now-unused `bare_name` variable.

**Cascading impacts found:**

- `tests/test_artifact_placement.py` — all bare-name assertions in T1–T7 will fail after the fix. Nine explicit assertions reference `folder / "request.md"`, `folder / "analysis.md"`, and `folder / "UAT_scenarios.md"`.

- `.aib_brain/conventions/analysis-convention.md` — section 3 (File Naming & Location) explicitly states: *"Archived phase — `analysis-<request_id>.md` is moved … to `.aib_memory/requests/<request-folder>/analysis.md` (bare name in the subfolder)"*. Must be updated to reflect ID-suffix preservation.

- `.aib_brain/conventions/request-convention.md` — "File Location & Naming (normative)" section states the same pattern for `request.md` and includes the note *"Only one `request.md` may exist per request folder (archived phase)"*. Both must be updated.

- `.aib_memory/context.md` (auto-generated) — The Component Map for "Move Artifacts Script" and "Request Artifacts" explicitly describe bare-named archived files. This is auto-generated by `aib-context.md`; it should be regenerated after the code fix is applied.

- `move-request-artifacts.py` docstring — the inline comment *"Archived artifacts keep bare names (e.g. `request.md`) inside the subfolder."* and the module-level docstring reference `request.md, analysis.md, and UAT_scenarios.md` as the moved names. Both must be updated.

**No other scripts depend on archived artifact names:**

`close-request.py` invokes `move_artifacts` as a black box (safety-net call) and does not reference the resulting filenames. The register's `folder` column continues to point to the folder path (unchanged). No other Python scripts in `.aib_brain/tools/` read archived artifact filenames.

### Industry Knowledge

**Finding 1 — Filename stability as a traceability principle (Software Configuration Management literature)**

A well-established SCM principle is that artifact identity should be immutable and traceable through its full lifecycle. Renaming a file at archival time breaks the direct filename-to-identity link that developers and tools rely on for search, diff, and audit. Tools like Git, Jira, and most artifact repositories preserve original filenames when archiving. Applicability: directly supports preserving the request ID in filenames — the archived `request-R-20260514-2159.md` is trivially traceable to the register entry, whereas `request.md` inside any arbitrary subfolder is not.

**Finding 2 — Idempotency and stateless naming (Unix tooling convention)**

Standard Unix archival tools (`tar`, `zip`, `rsync`) preserve source filenames by default unless explicitly instructed to rename. The convention of preserving source names prevents silent collisions and makes the archive self-describing. Applicability: the ID-suffixed filename makes each archive folder self-describing without requiring reference to the register — a `request-R-20260514-2159.md` file immediately communicates its identity. The bare-name convention offers a marginally simpler look at the cost of that traceability.

### Feasibility Notes

## Implementation Alternatives

### Alternative A — Preserve ID Suffix (ID-suffixed destination)

Use `filename` (already computed in the loop as `artifact_name(artifact_type, request_id)`) as the destination filename inside the archive subfolder. One-line change in `move-request-artifacts.py`. Update all bare-name references in tests, docstrings, and convention files.

- **Trade-offs:** Filenames in archive are longer but traceable without register lookup; convention files and test file require mechanical updates.

- **Expected codebase impact:** `move-request-artifacts.py` (1 line + 2 docstring updates), `tests/test_artifact_placement.py` (9 assertions), `analysis-convention.md` (2 sentences), `request-convention.md` (2 sentences), `context.md` (auto-regenerated).

### Alternative B — Keep Bare Names (no change)

Accept current behavior; update documentation to describe it as intentional.

- **Trade-offs:** Simpler filenames in archive; archived artifacts lose direct filename-to-identity link; contradicts explicit developer requirement.

- **Expected codebase impact:** None to code; documentation-only clarification.
