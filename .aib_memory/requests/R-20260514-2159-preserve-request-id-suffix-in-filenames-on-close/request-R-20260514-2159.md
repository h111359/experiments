## Goal

When closing a request, `request-<request_id>.md` and `analysis-<request_id>.md` are currently renamed to `request.md` and `analysis.md` (stripping the request ID suffix) as they are moved into the archived request folder. The developer wants this behavior changed: the request ID suffix MUST be preserved in the filenames — files should remain named `request-<request_id>.md` and `analysis-<request_id>.md` after archival.

## Background

The AIB lifecycle archives request artifacts on close by moving them from the `.aib_memory/` root into the request subfolder. The current tooling (likely `move-request-artifacts.py`) renames files to bare names (`request.md`, `analysis.md`) during this move. The developer considers this a defect: stripping the ID suffix makes archived files ambiguous and potentially harder to trace.

## Scope

- The `move-request-artifacts.py` script — specifically the logic that determines the destination filename when moving `request-<request_id>.md` and `analysis-<request_id>.md` to the request archive folder.

- Any other script or tool involved in the close lifecycle that renames these files (e.g., `close-request.py`).

- The `request-convention.md` file, which currently documents bare-name archival and may need updating to reflect the new behavior.

- Any documentation or test files that reference the bare-name convention (`request.md`, `analysis.md`) in the archived folder.

## Out of scope

- Renaming behavior for other artifact types (e.g., iteration logs, input archives).

- Changes to the active-phase naming rules (`request-<request_id>.md` at `.aib_memory/` root).

- The `create-request.py` or `finalize-input.py` lifecycle phases.

## Constraints

- The fix must be backwards-compatible: already-archived requests with bare names are not retroactively renamed.

- All automated tests must pass after the change.

## Success criteria

- After closing a request, the archived folder contains `request-<request_id>.md` and `analysis-<request_id>.md` (ID suffix preserved), not `request.md` and `analysis.md`.

- The `requests_register.md` `folder` column still points to the correct folder path.

- All existing automated tests pass, and any tests that verify close/move behavior are updated to assert the new filename pattern.

- `request-convention.md` reflects the updated archived-phase naming rule.

## Assumptions

- A1: The change applies going forward only; already-archived requests with bare-named files are not retroactively renamed.
  - Risk if false: Retroactive rename would require a migration script touching existing archive folders; significantly larger scope.

- A2: No external tooling (CI, scripts outside `.aib_brain/`) reads archived artifact filenames by their bare names (`request.md`, `analysis.md`).
  - Risk if false: External tools relying on bare names would break; additional search across CI workflow files would be needed.

- A3: The `test_close_request.py` suite does not independently assert archived artifact filenames (it was read and confirmed to not do so in T1–T7-equivalent assertions beyond what is in `test_artifact_placement.py`).
  - Risk if false: Additional test assertions would need updating.

#### Intent
Remove the bare-name renaming logic so archived artifacts retain their `<type>-<request_id>.md` filename.

#### Outputs
`.aib_brain/tools/move-request-artifacts.py` — modified

#### Procedure
In `.aib_brain/tools/move-request-artifacts.py`, locate the lines inside the `for artifact_type in _ARTIFACT_TYPES:` loop:

#### Done criteria
