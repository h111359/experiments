## Goal

Rename the per-request planning artifact from `request-<ID>.md` to `plan-<ID>.md` and reduce its mandatory section schema from ten sections to four execution-only directives (`## Goal`, `## Constraints`, `## Success criteria`, `## Plan`), with human-facing context sections relocated to a new `## Request Context` section in `analysis-<ID>.md`. Rename the governing convention file from `request-convention.md` to `plan-convention.md`.

## Background

The current `request-<ID>.md` artifact serves two audiences in a single file: human reviewers (who read `## Background`, `## Scope`, `## Out of scope` for project context and rationale) and the AI Automation Agent (which reads all sections to extract implementation directives). This dual-audience design creates a risk that the AI Automation Agent interprets human-narrative and contextual content as execution instructions, producing scope misinterpretations during implementation. The reform renames the artifact and reduces it to four execution-only sections, with context sections relocated to `analysis-<ID>.md`.
