## Goal

Add explicit, unconditional instructions to `aib-analysis.md` and `aib-implement.md` that prohibit reading closed (archived) request files during any phase of their execution. The rule must be stated clearly enough that an AI agent cannot rationalize an exception and must halt visibly if it detects it has read or is about to read a closed-request artifact.

## Background

Closed requests live in `.aib_memory/requests/<folder>/` subfolders and may contain stale, overwritten, or partially superseded instructions. Basing implementation or analysis decisions on these artifacts is dangerous: the content no longer represents authoritative scope and may contradict the current product state. The existing `GC-01` constraint in `aib-analysis.md` prohibits reading `inputs/input-archive-*.md` files, but no comparable blanket rule covers the other artifacts in closed request folders (e.g., `request-<id>.md`, `analysis-<id>.md`, `implementation-<id>.md`) that are stored there after archival.

## Scope

- Add a Global Constraint (or equivalent prominent rule block) to `aib-analysis.md` prohibiting reading any file inside `.aib_memory/requests/` subfolders that belong to a Closed request.

- Add the same prohibition to `aib-implement.md` under its "Rules" section.

- The rule must cover ALL artifact types (request, analysis, implementation, input archives, and any other file) inside a closed-request subfolder.

- The rule must apply to ALL phases / steps of both prompts (preflight, analysis generation, implementation, Q&A sub-flow, context enrichment, etc.).

- The prohibition language must be stated as a hard constraint — not a soft recommendation — so an AI agent treats it as a halting condition if violated.

## Out of scope

- Changes to `aib-context.md` or other prompts not listed above.

- Changes to Python tooling scripts.

- Changes to convention files other than any update needed to propagate the rule to `context.md`.

- Introducing a new constraint ID scheme beyond what already exists in each prompt.

## Constraints

- MUST NOT change the existing `GC-01` constraint wording; the new constraint is additive.

- The new prohibition MUST be phrased consistently across both prompts (same intent, adapted to each prompt's style).

- The constraint MUST be placed in a section of each prompt that is guaranteed to be read early (preflight or global constraints), not buried in an optional section.

- Do not alter the numbering of existing sections in either prompt; insert or append the new constraint without renumbering.

## Success criteria

- `aib-analysis.md` contains an explicit constraint stating that files inside Closed request subfolders MUST NOT be read during any phase of execution.

- `aib-implement.md` contains the same explicit constraint under its Rules section.

- The constraint language is unambiguous: it names the path pattern (`.aib_memory/requests/<folder>/` for Closed requests), lists the prohibited action (reading), and states the scope (all phases, all artifact types).

- `aib-analysis.md` existing `GC-01` remains unchanged.

- `context.md` is updated to reflect the new rule.

- All existing tests pass with no regressions.

## Assumptions

- A1: All closed-request artifacts (including `request-<id>.md`, `analysis-<id>.md`, and `implementation-<id>.md`) reside exclusively in `.aib_memory/requests/<folder>/` after `move-request-artifacts.py` runs; no closed-request artifact remains at `.aib_memory/` root.
  - Risk if false: The new GC path pattern would not cover stray artifacts at the root level, leaving a gap.

- A2: The existing GC naming scheme in `aib-analysis.md` (GC-01 through GC-05) uses sequential integer suffixes; the next available ID is GC-06.
  - Risk if false: ID collision or confusion; resolution is trivial (check the file before inserting).

- A3: No other prompt file (`aib-context.md`, `aib-initialize.md`, etc.) has an equivalent gap that is in scope; the request explicitly limits changes to `aib-analysis.md` and `aib-implement.md`.
  - Risk if false: The gap would remain in other prompts, but fixing them is out of scope for this request.
