## Goal

Add a content currency policy rule to `.aib_brain/prompts/aib-context.md` so that every generated `.aib_memory/context.md` reflects only the current product state — without version history annotations (e.g., "introduced in vX.Y.Z", "deprecated as of vX") and without glossary entries for deprecated or removed concepts. Then re-execute the updated prompt to regenerate `.aib_memory/context.md`, removing all existing violations.

## Constraints

- Only `.aib_brain/prompts/aib-context.md` and `.aib_memory/context.md` may be modified; no other files are in scope.
- The new rule in `aib-context.md` MUST be additive only — it must not alter or remove any existing behavioral rule.
- `.aib_brain/conventions/context-convention.md` MUST NOT be modified.
- The `context.md` update MUST be produced by executing the `aib-context.md` prompt (not by directly editing the file), to validate the rule in practice.
- Python 3.10+ and standard library only are available for any tool invocations.

## Success criteria

- `.aib_brain/prompts/aib-context.md` contains a clearly documented rule prohibiting version history annotations ("introduced in vX", "deprecated as of vX", "removed as of", "(Deprecated)" labels) in generated content.
- `.aib_brain/prompts/aib-context.md` contains a rule prohibiting glossary entries for deprecated or removed concepts.
- `.aib_memory/context.md` after regeneration contains no occurrences of: "introduced in v", "deprecated as of", "removed as of", "(Deprecated)", or request IDs embedded in glossary definitions.
- `.aib_memory/context.md` after regeneration does not contain an `Iteration` glossary entry.
- All automated tests pass after the changes.

## Plan

### Task 1: Add content currency rule to `aib-context.md`

#### Intent
Augment the synthesis phase of `.aib_brain/prompts/aib-context.md` with an explicit content currency policy rule that prohibits version history annotations and deprecated-concept entries in the generated output.

#### Outputs
`.aib_brain/prompts/aib-context.md` — modified with the new synthesis constraint rule inserted into Phase 4 — Synthesis.

#### Procedure
Read `.aib_brain/prompts/aib-context.md` in full to identify the exact insertion point within Phase 4 — Synthesis (or Core requirements), where a new "Content currency rule" paragraph fits naturally alongside existing synthesis constraints.

Insert the following rule as a named block in the synthesis phase (after the existing "Evidence-backed synthesis rules" or within Phase 4, before the section on mandatory sections):

```
Content currency rule: All content MUST reflect the current state of the product only.
MUST NOT include version history annotations such as "introduced in vX.Y.Z", "added in vX",
"deprecated as of vX", "removed as of", or "(Deprecated)" labels. Glossary entries for
deprecated or removed concepts MUST NOT be included; only currently active concepts and
terms belong in the glossary. Describe what currently exists and is active; historical
change information belongs in changelogs and version logs, not in context.md.
```

Verify the edit by re-reading the modified section of `.aib_brain/prompts/aib-context.md` to confirm the rule is present and correctly formatted.

#### Done criteria
- `.aib_brain/prompts/aib-context.md` contains the content currency rule in Phase 4 or Core requirements.
- No existing rule has been altered or removed.
- The file remains valid Markdown.

#### Dependencies
None.

#### Risk notes
`.aib_brain/` modification is authorized by this active request per AIB convention. The rule wording must be unambiguous enough to prevent the AI agent from retaining any version annotation under an alternate phrasing.

### Task 2: Execute `aib-context.md` to regenerate `context.md`

#### Intent
Run the updated `aib-context.md` prompt to produce a new `.aib_memory/context.md` that applies the content currency rule and removes all existing version history violations.

#### Outputs
`.aib_memory/context.md` — fully replaced with version-history-free content.

#### Procedure
Execute `.aib_brain/prompts/aib-context.md` as a prompt action. The prompt performs its own Phase 1–6 cycle (preflight, supplementary read, workspace scan, synthesis, write, post-write validation).

During synthesis, apply the content currency rule to every glossary entry:
- Remove the phrase "introduced in v1.2.0" from the `Auto-request` glossary entry, retaining the behavioral description of what auto-request does.
- Remove the entire `Iteration` glossary entry (deprecated concept; no current-state representation).
- Remove the embedded request ID reference "(R-20260422-1308)" from the `instructions.md` glossary entry, retaining the capability description.
- Scan all other glossary entries and any bullet content for additional version annotations; remove any found.

After the prompt completes writing `.aib_memory/context.md`, confirm the file has been fully replaced.

#### Done criteria
- `.aib_memory/context.md` is non-empty and valid Markdown.
- No occurrence of "introduced in v", "deprecated as of", "removed as of", "(Deprecated)", or embedded request IDs in glossary definitions.
- No `Iteration` glossary entry is present.
- All mandatory sections defined in `context-convention.md` are present in the correct order.

#### Dependencies
Task 1 (content currency rule must be in place before executing the prompt).

#### Risk notes
The prompt regenerates the entire file — it replaces, not appends. If the workspace file inventory has changed since the last run, the new `context.md` may reflect those changes as well (expected behavior, not a risk). Verify all mandatory sections are present after regeneration.
