### F-011 · **High** · instructions.md exception for README.md editing is undeclared in aib-implement.md safety rules — creates a hidden override that could confuse agents

  > "Do not add files under .aib_brain"
- instructions.md, README maintenance directive:

**Why it matters:**
The safety rule in aib-implement.md is stated without any exception clause. An agent reading the prompt in isolation would refuse to touch README.md. An agent reading instructions.md first (as required by step 1) would understand the exception. This gap creates an inconsistency between the safety model stated in the prompt and the behavioral override in instructions.md. **Recommendation:**

> "Exception: README.md MAY be reviewed and updated after each implement run to align with context.md (see workspace instructions directive)."

**Risk/Tradeoffs:** Medium — modifying a safety section requires care; adding an explicit exception is safer than leaving it implicit.
