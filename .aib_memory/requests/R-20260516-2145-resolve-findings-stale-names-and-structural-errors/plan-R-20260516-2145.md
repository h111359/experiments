## Goal

Correct 13 identified inconsistencies across AIB framework files arising from three completed rename operations (`aib-analysis.md` → `aib-analyze.md`, `aib-context.md` → `aib-refresh-context.md`, `request.md` → `plan-<request_id>.md`). Findings cover stale cross-references in prompts, conventions, tool scripts, and `context.md`; conflicting mandatory-section counts; a duplicate subsection number in `analysis-convention.md`; a dangling "Phase 3" label in `aib-refresh-context.md`; an undeclared safety exception in `aib-implement.md`; a minimum-findings threshold discrepancy; and an undocumented artifact type in `move-request-artifacts.py`. All 13 findings (F-001 through F-012, F-015) must be corrected in the identified target files.

- All fixes are string-level substitutions or minor structural edits; tool script logic MUST NOT be changed.
- F-003 resolution (Q001 answered, Option A chosen): Replace both occurrences of "Phase 3" with "Phase 2" in `.aib_brain/prompts/aib-refresh-context.md` (preamble and section 4.3 only). The Reverse-Engineering Evidence Collection section remains unlabeled as a Phase 2 addendum.

- Stale `aib-analysis.md` references in test assertion message strings in `tests/test_analysis_prompt_structure.py` are explicitly out of scope.
- `.aib_brain/README.md` may be modified if discrepancies against `context.md` are found, per `instructions.md` README maintenance directive.
- `logs/next_version_changes.md` MUST be appended with change bullets per `instructions.md` directive.

## Success criteria
