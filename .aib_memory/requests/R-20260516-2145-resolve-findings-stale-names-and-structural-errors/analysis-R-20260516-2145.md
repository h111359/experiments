### Industry Knowledge

**Finding 1 — Docs-as-code rename hygiene (source: DIATAXIS documentation framework and general software engineering practice):**
When a file is renamed in a docs-as-code system, all inbound cross-references must be updated atomically in the same commit or immediately subsequent commits. Allowing stale references to persist creates a "broken link" problem equivalent to dead hyperlinks in web documentation — the reference exists but points to nothing. This is particularly acute when the referencing document is executed by a non-human agent that cannot infer corrections. Applicability to this request: high — all seven stale-name findings (F-004 through F-010) are exactly this class of defect.

**Finding 2 — Normative document consistency (source: ISO/IEC/IEEE 29148:2018 Systems and Software Engineering — Life Cycle Processes — Requirements Engineering):**
Standards bodies define the "consistency" requirement attribute: a document set is consistent when no two documents contradict each other. Numeric inconsistencies in mandatory-section counts (F-001, F-012) and numbering-order mismatches (F-002) are direct violations of this attribute. When an agent executes against an inconsistent constraint set, it must resolve the contradiction through inference, producing non-deterministic output. Applicability to this request: high — fixing the counts and numbering eliminates the inference requirement.

**Finding 3 — Implicit override fragility (source: software architecture principle of explicit contracts, widely documented in "Designing Software Architectures" by Cervantes and Kazman, and Martin's Clean Architecture):**
Safety rules that can be silently overridden by a configuration file create a fragile architecture. The override survives only as long as the configuration file is intact and is read before the rule. Making the exception explicit in the rule document (aib-implement.md) eliminates this single point of failure and makes the safety model self-documenting. Applicability to this request: high — F-011 is a textbook example of this pattern.
